## 1. 전체 사용자 여정 프레임워크

사용자의 여정은 5단계로 구성됩니다.

```
┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│ 1. 인지   │ → │ 2. 탐색   │ → │ 3. 실행   │ → │ 4. 활용   │ → │ 5. 재방문 │
│ Awareness│   │ Explore  │   │ Execute  │   │ Apply    │   │ Return   │
│          │   │          │   │          │   │          │   │          │
│ "이런게   │   │ "어떤 API │   │ "지금     │   │ "내 프로  │   │ "다른 API│
│  있구나"  │   │  가 나한테│   │  세팅     │   │  젝트에   │   │  도 해봐  │
│          │   │  맞지?"   │   │  해보자"  │   │  적용하자"│   │  야지"   │
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
```

## 2. 단계별 상세 여정

### 2.1 Stage 1: 인지 (Awareness)

**"클라우드 API를 무료로 쓸 수 있다는 걸 알게 되는 단계"**

| 항목 | 내용 |
|------|------|
| 사용자 상태 | 특정 기능이 필요하지만 어떻게 구현할지 모름 |
| 트리거 | "이미지에서 텍스트 추출하고 싶어" / "무료 번역 API 없나?" |
| 채널 | Google 검색, SNS, 개발자 커뮤니티, 유튜브 |
| 감정 | 호기심 + 약간의 불안감 ("나도 할 수 있을까?") |

**터치포인트 & 우리의 대응:**

```
[검색] "구글 번역 API 무료"
    ↓
[결과] CloudSetup.guide 가이드 페이지
    ↓
[랜딩] 가이드 상단에서 즉시 확인 가능:
    • ⏱ 예상 소요시간: 15분
    • 💰 무료 한도: 월 50만자
    • ⭐ 난이도: 초급
    • ✅ 마지막 검증일: 2025.01.15
    ↓
[반응] "오, 15분이면 되고 무료네? 해볼까?"
```

**핵심 지표 (KPI):**
- 유기 검색 유입 수 (Organic Search Traffic)
- 검색 키워드 순위 (SERP Ranking)
- 랜딩 페이지 이탈률 (Bounce Rate) — 목표: 40% 이하
- 첫 페이지 체류 시간 — 목표: 3분 이상

**최적화 포인트:**
- SEO: 타겟 키워드가 제목, H1, 메타 디스크립션, URL에 포함
- 페이지 상단에 "이 가이드로 할 수 있는 것"을 즉시 보여줌
- "무료", "~분 소요", "스크린샷 포함" 등의 신뢰 시그널
- 과금 공포 해소: "무료 한도 안에서 진행됩니다" 명시

### 2.2 Stage 2: 탐색 (Explore)

**"내 상황에 맞는 API와 가이드를 찾는 단계"**

| 항목 | 내용 |
|------|------|
| 사용자 상태 | "무료 API가 많네... 나한테 뭐가 맞지?" |
| 행동 | 플랫폼별 탐색, 카테고리 필터링, 비교 페이지 확인 |
| 감정 | 선택의 어려움 + 정보 과부하 |

**사용자 흐름:**

```
방법 A: 목적에서 출발
┌───────────────────────────────────┐
│  "무엇을 만들고 싶으세요?"          │
│                                   │
│  🖼️ 이미지 분석    🌐 웹사이트     │
│  🗣️ 음성 인식     💬 번역         │
│  🗺️ 지도 서비스    💾 데이터베이스  │
│  🤖 AI 챗봇       📦 파일 저장     │
└───────────────────────────────────┘
         ↓ "이미지 분석" 선택
┌───────────────────────────────────┐
│  이미지 분석에 사용할 수 있는 무료 API │
│                                   │
│  ⭐ Google Vision API              │
│     무료: 월 1,000회 | 초급 | 15분   │
│                                   │
│  ● AWS Rekognition                │
│     무료: 월 5,000회 (1년) | 중급    │
│                                   │
│  ● Azure Computer Vision          │
│     무료: 월 5,000회 | 중급          │
│                                   │
│  [⚖️ 세 서비스 비교하기]             │
└───────────────────────────────────┘

방법 B: 플랫폼에서 출발
┌───────────────────────────────────┐
│  플랫폼 선택                       │
│                                   │
│  ☁️ GCP (15 APIs)                  │

│  🔵 Azure (10 APIs)               │
│  ⚡ Supabase (5 가이드)            │
│  ▲ Vercel (3 가이드)               │
└───────────────────────────────────┘

방법 C: 직접 검색
┌───────────────────────────────────┐
│  🔍 "vision api"                   │
│                                   │
│  검색 결과:                        │
│  • GCP Vision API 설정 가이드       │
│  • AWS Rekognition 설정 가이드      │
│  • GCP vs AWS 이미지 분석 비교      │
└───────────────────────────────────┘
```

**핵심 지표 (KPI):**
- 탐색 페이지에서 가이드 페이지로의 전환율 — 목표: 30% 이상
- 검색 사용률
- 필터 사용률
- 탐색 중 이탈률 — 목표: 60% 이하

**최적화 포인트:**
- 목적별/플랫폼별/난이도별 다중 탐색 경로 제공
- 각 API 카드에 "무료 한도", "난이도", "소요 시간" 즉시 표시
- "가장 인기 있는 가이드" 하이라이트로 선택 부담 감소
- 비교 페이지로 자연스럽게 유도

### 2.3 Stage 3: 실행 (Execute)

**"실제로 API를 세팅하는 단계 — 가장 중요한 단계"**

| 항목 | 내용 |
|------|------|
| 사용자 상태 | 가이드를 보면서 하나씩 따라하는 중 |
| 행동 | 스크린샷 확인 → 클릭 → 다음 단계 반복 |
| 감정 | 집중 + 불안 ("이게 맞나?") + 성취감 (단계 완료 시) |

**상세 플로우 (GCP Vision API 세팅 예시):**

```
Step 0: 선행 확인
┌─────────────────────────────────────┐
│  ⚠️ 이 가이드를 시작하기 전에:        │
│                                     │
│  ✅ GCP 계정이 있으신가요?             │
│     [네, 있어요] → Step 3로 건너뛰기  │
│     [아니요] → GCP 계정 만들기 가이드  │
│                                     │
│  ✅ 결제 계정을 설정하셨나요?           │
│     [네] → Step 3로 건너뛰기          │
│     [아니요] → 결제 계정 설정 가이드    │
│                                     │
│  💡 결제 계정을 등록해도 무료 한도      │
│     안에서는 과금되지 않습니다!         │

          ↓
Step 1: 프로젝트 만들기 (2분)
┌─────────────────────────────────────┐
│  [스크린샷: GCP 콘솔 상단]            │
│  🔴 "프로젝트 선택" 클릭 표시          │
│                                     │
│  1. GCP 콘솔 상단의 프로젝트 선택을    │
│     클릭하세요.                       │
│  2. "새 프로젝트"를 클릭하세요.        │
│  3. 프로젝트 이름을 입력하세요.        │
│     (예: "my-vision-project")       │
│  4. "만들기"를 클릭하세요.             │
│                                     │
│  ⚠️ 주의: 프로젝트 ID는 나중에        │
│     변경할 수 없으니 신중하게 정하세요.  │
│                                     │
│  ✅ 프로젝트가 생성되었나요?            │
│     [네, 다음으로] [막혔어요 — 도움말]  │
└─────────────────────────────────────┘

Step 2: Vision API 활성화 (3분)
┌─────────────────────────────────────┐
│  [스크린샷: API 라이브러리 화면]       │

│                                     │

          ↓

          ↓
```

**핵심 지표 (KPI):**
- 가이드 완료율 — 목표: 60% 이상
- 단계별 이탈율 (어디서 가장 많이 포기하는지)
- 평균 완료 시간 vs 예상 시간
- "막혔어요" 클릭 빈도 (문제 지점 파악)
- 피드백 점수 (도움됨/어려웠어요)

**최적화 포인트:**
- 매 단계마다 "성공 확인" 체크포인트
- 과금 공포를 지속적으로 해소하는 문구
- "막혔어요" 버튼 → 자주 발생하는 문제 FAQ / 댓글 섹션
- 진행률 바로 동기부여
- 중간에 이탈해도 다시 돌아왔을 때 이어서 할 수 있도록 (진행률 저장 — Phase 3)

### 2.4 Stage 4: 활용 (Apply)

**"세팅한 API를 실제 프로젝트에 적용하는 단계"**

| 항목 | 내용 |
|------|------|
| 사용자 상태 | API 키를 받았고, 이제 뭘 할 수 있는지 알고 싶음 |
| 행동 | 활용 예시 확인, 코드 복붙, 자기 프로젝트에 적용 |
| 감정 | 성취감 + 가능성의 설렘 |

**활용 콘텐츠 구조:**

```
Vision API 세팅 완료 후 →

활용 가이드 1: 이미지에서 텍스트 추출(OCR)하기
├── 사용 시나리오: "명함 사진으로 연락처 자동 저장"
├── 코드 예시 (Python, Node.js)
├── 실행 결과 스크린샷
└── 응용 아이디어

활용 가이드 2: 이미지 라벨링 자동화
├── 사용 시나리오: "쇼핑몰 상품 이미지 자동 태깅"
├── 코드 예시
├── 무료 한도 내 처리 가능 분량
└── 응용 아이디어

활용 가이드 3: 유해 콘텐츠 감지
├── 사용 시나리오: "커뮤니티 이미지 필터링"
├── 코드 예시
└── 주의사항
```

**핵심 지표 (KPI):**
- 세팅 가이드 → 활용 가이드 이동률 — 목표: 25% 이상
- 활용 가이드 코드 복사 횟수
- 활용 가이드 피드백 점수

### 2.5 Stage 5: 재방문 (Return)

**"다른 API도 세팅하러, 또는 업데이트 확인하러 돌아오는 단계"**

| 항목 | 내용 |
|------|------|
| 사용자 상태 | 한 API 세팅에 성공, 다른 것도 하고 싶음 |
| 트리거 | 새 프로젝트, 기존 프로젝트 확장, 무료 한도 변경 소식 |
| 감정 | 신뢰 + 편안함 ("여기 오면 된다") |

**재방문 유도 메커니즘:**

```
1. 가이드 완료 시 "다음으로 추천" 제안
   "Vision API를 세팅하셨으니, 이것도 해보세요:"
   → Translation API (번역)
   → Speech-to-Text (음성 인식)
   → Cloud Storage (이미지 저장)

2. 무료 한도 변경 알림 (뉴스레터 — Phase 3)
   "GCP Cloud Run 무료 한도가 변경되었습니다"

3. 유틸리티 도구 (재방문 유도)
   → 무료 한도 계산기 (매번 확인 필요)
   → API 키 관리 체크리스트

4. SEO를 통한 자연 재유입
   → 새 API 세팅이 필요할 때 검색 → 다시 우리 사이트

5. 커뮤니티 알림 (Phase 3)
   → "내가 질문한 글에 답변이 달렸습니다"
```

**핵심 지표 (KPI):**
- 재방문률 — 목표: 20% 이상
- 사용자당 완료 가이드 수 — 목표: 2개 이상
- 뉴스레터 구독률 — 목표: 방문자의 3%
- 북마크/즐겨찾기 비율

## 3. 이탈 포인트 & 대응 전략

### 3.1 주요 이탈 지점

| 단계 | 이탈 지점 | 이탈 이유 | 대응 |
|------|----------|----------|------|
| 인지 | 랜딩 페이지 | "이거 나한테 맞는 건지 모르겠어" | 상단에 타겟 사용자 명시, 난이도 표시 |
| 인지 | 랜딩 페이지 | "영어네, 나가자" | hreflang + 한국어 즉시 확인 가능 |
| 탐색 | 가이드 목록 | "너무 많아서 뭘 선택할지 모르겠어" | 목적별 추천, 인기순 정렬 |
| 실행 | 결제 정보 입력 | "신용카드를 넣으라고? 과금되면?" | 빨간 박스 안내: "무료 한도 내 과금 없음" |
| 실행 | CLI 명령어 | "터미널? 나 그런 거 모르는데" | 웹 UI 방법을 기본으로, CLI는 추가 옵션 |
| 실행 | 스크린샷 불일치 | "내 화면이랑 다른데?" | 최종 검증일 표시, 피드백 버튼 |
| 실행 | 에러 발생 | "에러 메시지가 나왔어" | 흔한 에러 + 해결법 FAQ |
| 활용 | 코드 실행 | "코드를 어디서 실행하는 건데?" | 실행 환경 안내 (Google Colab 등) |
| 재방문 | — | "이미 했으니까 다시 올 이유가 없어" | 다음 가이드 추천 + 유틸리티 도구 |

### 3.2 핵심 이탈 방지 UI 패턴

**패턴 1: 과금 공포 해소 배너**
```
┌─────────────────────────────────────┐
│ 💰 과금 걱정 마세요!                  │
│ 이 가이드의 모든 단계는 무료 한도      │
│ 안에서 진행됩니다.                    │
│ 실수로 과금되지 않도록 예산 알림       │
│ 설정도 함께 안내해 드려요.            │
└─────────────────────────────────────┘
```

**패턴 2: 진행 확인 체크포인트**
```
✅ 이 화면이 보이시나요?
   [스크린샷]
   [네, 보여요 → 다음]
   [다르게 보여요 → 트러블슈팅]
```

**패턴 3: 선행 가이드 안내**
```
⚠️ 이 단계를 진행하려면 GCP 계정이 필요합니다.
   [이미 있어요 → 건너뛰기]
   [없어요 → GCP 계정 만들기 (5분)]
```

## 5. 측정 및 분석 계획

### 5.1 퍼널 분석

```
유입 (100%) → 탐색 (60%) → 가이드 시작 (40%) → 가이드 완료 (25%) → 재방문 (15%)
                 ↑              ↑                    ↑                  ↑
             이탈 40%        이탈 20%            이탈 15%           이탈 10%
```

### 5.2 추적할 이벤트 (GA4)

| 이벤트 | 트리거 | 용도 |
|--------|--------|------|
| `page_view` | 페이지 로드 | 기본 트래픽 분석 |
| `guide_start` | 가이드 첫 단계 조회 | 전환율 분석 |
| `guide_step` | 각 단계 이동 | 단계별 이탈 분석 |
| `guide_complete` | 마지막 단계 도달 | 완료율 분석 |
| `guide_stuck` | "막혔어요" 클릭 | 문제 지점 파악 |
| `guide_feedback` | 피드백 제출 | 품질 측정 |
| `code_copy` | 코드 복사 버튼 클릭 | 활용도 측정 |
| `search` | 사이트 내 검색 | 사용자 니즈 파악 |
| `filter_use` | 필터 사용 | 탐색 패턴 분석 |
| `outbound_click` | 외부 링크 클릭 | 제휴 전환 추적 |
| `newsletter_subscribe` | 구독 신청 | 리텐션 지표 |

### 5.3 핵심 대시보드 구성

```
[일간]
• 총 방문자 수, 페이지뷰
• 가이드 시작/완료 수
• 인기 가이드 Top 5

[주간]
• 가이드별 완료율 변화
• "막혔어요" 빈도 Top 5 단계
• 검색 키워드 Top 10

[월간]
• 전체 퍼널 전환율
• 재방문률
• 광고 수익
• 콘텐츠 커버리지 (새로 추가된 가이드 수)
```
