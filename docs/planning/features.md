# ⚙️ Features Specification — CloudSetup.guide

> 최종 수정: 2025.02.07
> 버전: v1.0

---

## 1. 기능 우선순위 프레임워크

모든 기능은 **MoSCoW 방법론**으로 분류합니다.

| 분류 | 의미 | Phase |
|------|------|-------|
| **Must** | 없으면 출시 불가 | Phase 1 (MVP) |
| **Should** | 있어야 경쟁력 확보 | Phase 2 |
| **Could** | 있으면 좋지만 없어도 됨 | Phase 3 |
| **Won't** | 지금은 안 함 (향후 재검토) | Phase 4+ |

## 2. Phase 1: MVP 기능 (Must Have)

### F1.1 📖 Step-by-Step 가이드 시스템

**기능 설명:** MDX 기반의 단계별 가이드 렌더링 시스템

**상세 요구사항:**

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F1.1.1 | MDX 파일에서 가이드 콘텐츠 파싱 및 렌더링 | Must |
| F1.1.2 | Step 컴포넌트: 번호, 제목, 예상 시간 표시 | Must |
| F1.1.3 | Screenshot 컴포넌트: 이미지 + 하이라이트 오버레이 | Must |
| F1.1.4 | Callout 컴포넌트: warning, info, danger, tip 4가지 타입 | Must |
| F1.1.5 | CopyBlock 컴포넌트: 코드/명령어 원클릭 복사 | Must |
| F1.1.6 | FreeTierInfo 컴포넌트: 무료 한도 배지 표시 | Must |
| F1.1.7 | DevTip 컴포넌트: 개발자용 추가 정보 (접힌 상태) | Must |
| F1.1.8 | 가이드 상단 메타 정보: 소요시간, 난이도, 무료한도, 검증일 | Must |
| F1.1.9 | 이전/다음 단계 네비게이션 버튼 | Must |
| F1.1.10 | 목차(TOC): 현재 단계 하이라이트 + 클릭 이동 | Must |

**수용 기준 (Acceptance Criteria):**
- MDX 파일 추가만으로 새 가이드 페이지가 자동 생성되어야 함
- 스크린샷 하이라이트가 좌표 기반으로 빨간 박스를 오버레이해야 함
- 코드 복사 버튼 클릭 시 클립보드에 복사되고 "복사됨" 피드백 표시
- 모바일에서 스크린샷이 확대/축소(pinch-zoom) 가능해야 함

### F1.2 📊 진행률 표시

**기능 설명:** 가이드 진행 상황을 시각적으로 표시

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F1.2.1 | 상단 진행률 바 (현재 단계 / 전체 단계) | Must |
| F1.2.2 | 단계별 도트 네비게이션 (클릭으로 이동) | Must |
| F1.2.3 | 완료된 단계 시각적 구분 (체크 아이콘) | Must |

**수용 기준:**
- 진행률 바가 스크롤 시에도 상단에 고정(sticky)되어야 함
- 단계 도트 클릭 시 해당 단계로 스무스 스크롤

### F1.3 🧭 선행 가이드 체크

**기능 설명:** 현재 가이드를 시작하기 전에 필요한 선행 단계를 안내

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F1.3.1 | 가이드 시작 전 선행 가이드 체크리스트 표시 | Must |
| F1.3.2 | "이미 완료" / "아직 안 함" 선택 옵션 | Must |
| F1.3.3 | "아직 안 함" 선택 시 해당 선행 가이드로 링크 | Must |
| F1.3.4 | "이미 완료" 선택 시 현재 가이드 바로 시작 | Must |

**수용 기준:**
- meta.json의 prerequisites 배열에서 자동으로 생성
- 선행 가이드가 없는 경우 (예: GCP 계정 만들기) 체크리스트 미표시

### F1.4 🔍 검색 기능

**기능 설명:** 사이트 내 가이드 검색

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| F1.4.1 | 홈페이지 메인 검색바 | Must |
| F1.4.2 | 글로벌 헤더 검색 아이콘 (모든 페이지) | Must |
| F1.4.3 | 실시간 자동완성 (제목, 태그 기반) | Must |
| F1.4.4 | 검색 결과 페이지: 관련도순 정렬, 메타 정보 표시 | Must |

**기술 구현:** Pagefind (정적 사이트 검색 엔진, 빌드 시 인덱스 생성)

**수용 기준:**
- 검색 인덱스가 빌드 시 자동 생성
- 한국어/영어 각각 검색 가능
- 검색 결과에 가이드 제목, 플랫폼, 난이도, 무료한도 표시
- 검색 대기시간 200ms 이하

## 7. 비기능 요구사항 (Non-Functional Requirements)

### 7.1 성능

| 항목 | 요구사항 |
|------|---------|
| 초기 로드 시간 | < 2초 (LCP) |
| 인터랙션 응답 | < 100ms (FID) |
| 레이아웃 안정성 | < 0.1 (CLS) |
| 검색 응답 시간 | < 200ms |
| 이미지 로딩 | Lazy loading + blur placeholder |

### 7.2 보안

| 항목 | 요구사항 |
|------|---------|
| HTTPS | 필수 (Vercel 기본 제공) |
| CSP | Content Security Policy 헤더 설정 |
| XSS 방지 | MDX 렌더링 시 HTML 이스케이프 |
| 댓글 스팸 방지 | Rate limiting + CAPTCHA (Phase 3) |

### 7.3 가용성

| 항목 | 요구사항 |
|------|---------|
| 업타임 | 99.9% (Vercel SLA) |
| CDN | 글로벌 CDN (Vercel Edge Network) |
| 에러 페이지 | 커스텀 404, 500 페이지 |

### 7.4 확장성

| 항목 | 요구사항 |
|------|---------|
| 가이드 추가 | MDX 파일 추가만으로 가능 |
| 플랫폼 추가 | 설정 파일 추가만으로 가능 |
| 언어 추가 | 로케일 폴더 + 번역 파일 추가로 가능 |
