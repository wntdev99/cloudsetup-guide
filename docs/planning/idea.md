## 1. 한 줄 요약

**"비개발자도 따라할 수 있는, 클라우드 API 세팅의 원스톱 허브"**

---

## 2. 아이디어의 출발점

### 2.1 관찰된 문제

클라우드 플랫폼(GCP, AWS, Azure 등)은 매우 관대한 무료 티어를 제공합니다. GCP만 해도 Vision API 월 1,000회, Translation API 월 50만자, Cloud Run 월 200만 요청 등을 무료로 쓸 수 있습니다. 하지만 실제로 이 무료 API를 활용하는 사람은 극소수입니다.

그 이유는 단순합니다:

1. **"무료라는 건 알겠는데, 어떻게 세팅하는지 모르겠다"** — 공식 문서는 영어이고, 개발자 관점으로 쓰여 있어서 비개발자에게는 외국어나 다름없습니다.

2. **"어디서부터 시작해야 하는지 모르겠다"** — GCP 계정 만들기 → 결제 설정 → 프로젝트 생성 → API 활성화 → API 키 생성 → 테스트 호출까지, 선행 단계가 많은데 이 흐름을 한눈에 보여주는 곳이 없습니다.

3. **"실수로 요금 폭탄 맞을까 무섭다"** — 무료 한도가 있다는 건 알지만, 정확히 어디까지가 무료인지, 한도를 넘기면 어떻게 되는지 명확하지 않아 시도 자체를 포기합니다.

### 2.2 핵심 인사이트: "deg to rad 변환기" 모델

인터넷에서 "deg to rad"를 검색하면, 첫 번째로 나오는 웹사이트에서 바로 변환합니다. 사실 계산기로 할 수도 있고, AI에게 물어볼 수도 있지만, 그냥 검색해서 첫 번째 사이트를 씁니다. 이유는:

- **편의성:** 이미 거기 있으니까
- **습관:** 한 번 쓰면 다음에도 같은 사이트를 씀
- **신뢰:** "이 사이트는 항상 정확했으니까"

CloudSetup.guide는 이 모델을 그대로 적용합니다. "GCP Vision API 세팅 방법"을 검색하면 항상 우리 사이트가 나오고, 한 번 쓴 사용자는 다른 API 세팅이 필요할 때도 다시 찾아옵니다.

### 2.3 왜 지금인가?

- **AI 시대의 역설:** ChatGPT, Claude 같은 AI가 보편화되면서, 비개발자들도 "AI API를 직접 써보고 싶다"는 욕구가 폭발적으로 증가했습니다. 하지만 API를 세팅하는 과정은 여전히 어렵습니다.
- **클라우드 무료 티어 확대:** 모든 클라우드 벤더가 경쟁적으로 무료 한도를 늘리고 있습니다. GCP는 Cloud Run, AWS는 Lambda, Azure는 Functions 등 서버리스 무료 티어가 계속 확장 중입니다.
- **노코드/로우코드 트렌드:** Notion, Zapier, Make 등 비개발자도 자동화를 구축하는 트렌드가 가속화되고 있고, 이들이 "다음 단계"로 클라우드 API를 직접 호출하려 할 때 우리 사이트가 필요합니다.

## 3. 아이디어 핵심 구조

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│   사용자: "GCP Vision API로 이미지 분석하고 싶어"      │
│                                                      │
│   ┌──────────┐    ┌──────────┐    ┌──────────┐      │
│   │ 1. 탐색   │ →  │ 2. 세팅   │ →  │ 3. 활용   │      │
│   │          │    │          │    │          │      │
│   │ 무료한도  │    │ Step-by  │    │ 실전예시  │      │
│   │ 비교/검색 │    │ -Step    │    │ 템플릿   │      │
│   │ 목적별   │    │ 스크린샷  │    │ 조합추천  │      │
│   │ 탐색     │    │ 가이드   │    │          │      │
│   └──────────┘    └──────────┘    └──────────┘      │
│                                                      │
│   수익: [AdSense 광고] + [제휴 마케팅] + [향후 프리미엄]  │
│                                                      │
└──────────────────────────────────────────────────────┘
```

### 3.1 핵심 가치 제안 (Value Proposition)

| 기존 상황 | CloudSetup.guide |
|-----------|-----------------|
| 공식 문서는 영어 + 개발자용 | 한/영 동시, 비개발자도 따라할 수 있는 수준 |
| 스크린샷 없이 텍스트만 | 매 클릭마다 스크린샷 + 빨간 박스 하이라이트 |
| API별로 다른 사이트에서 검색 | 모든 클라우드 API를 한 곳에서 |
| 무료 한도가 정확히 얼마인지 불명확 | 무료 한도 대시보드로 한눈에 비교 |
| "어디서부터?"에 대한 답이 없음 | 선행/후행 가이드 로드맵 제공 |
| UI가 바뀌면 가이드가 무용지물 | 최종 검증일 표시 + 지속 업데이트 |

### 3.2 핵심 차별점

1. **스크린샷 기반 Step-by-Step:** AI가 아직 대체하지 못하는 영역입니다. 실시간으로 최신 GCP/AWS 콘솔 스크린샷을 제공하는 것은 AI가 할 수 없습니다.

2. **무료 한도 대시보드:** 모든 클라우드의 무료 티어 정보를 한 곳에 모아 비교하는 서비스는 아직 없습니다.

3. **가이드 의존성 로드맵:** "Vision API를 쓰려면 먼저 이것부터 해야 해요"를 시각적으로 보여주는 기능입니다.

4. **한/영 동시 운영:** 한국어로 국내 트래픽, 영어로 글로벌 트래픽을 동시에 확보합니다. 영어 기술 콘텐츠의 광고 RPM이 한국어의 3-5배이므로 수익성도 높습니다.

## 4. 범위 (Scope)

### 4.1 커버할 플랫폼 (초기 → 확장)

**Phase 1 (MVP):**
- Google Cloud Platform (GCP) — 핵심 10개 API

**Phase 2:**
- Amazon Web Services (AWS)
- Supabase
- Vercel

**Phase 3:**
- Microsoft Azure
- Cloudflare
- Firebase
- Netlify
- Railway
- PlanetScale / Neon

### 4.2 커버할 카테고리

- AI/ML: Vision, Translation, Speech-to-Text, Text-to-Speech, Gemini 등
- 컴퓨팅/서버리스: Cloud Run, Lambda, Functions, App Engine 등
- 데이터베이스: Firestore, DynamoDB, Supabase DB, PlanetScale 등
- 스토리지: Cloud Storage, S3, R2 등
- 인증: Firebase Auth, Supabase Auth, Cognito 등
- 지도: Google Maps Platform, Mapbox 등
- CDN/배포: Vercel, Cloudflare Pages, Netlify 등
- 모니터링/로깅: Cloud Logging, CloudWatch 등

### 4.3 커버하지 않는 것

- 엔터프라이즈급 아키텍처 설계
- 쿠버네티스/컨테이너 오케스트레이션 심화
- 멀티 클라우드 인프라 구축
- 특정 프로그래밍 언어 튜토리얼 (API 호출 예시는 제공하되, 언어 자체를 가르치지는 않음)

## 6. 성공 기준

### 6.1 단기 (6개월)

- GCP 핵심 가이드 10개 + AWS 핵심 가이드 5개 완성
- 월 5-10만 페이지뷰 달성
- Google 검색에서 주요 키워드 1페이지 진입 5개 이상
- AdSense 승인 및 월 10만원 이상 수익

### 6.2 중기 (1년)

- 전체 가이드 50개 이상 (5개 플랫폼)
- 월 20-30만 페이지뷰
- 월 수익 100만원 이상 (광고 + 제휴)
- 커뮤니티 댓글/질문 활성화

### 6.3 장기 (2년)

- 전체 가이드 100개 이상 (8개+ 플랫폼)
- 월 50-80만 페이지뷰
- 월 수익 300만원 이상
- 오픈소스 기여자 커뮤니티 형성
- "클라우드 API 세팅" 하면 떠오르는 브랜드

## 8. 왜 이것을 하는가? (개인적 동기)

로보틱스 엔지니어로서 클라우드 API를 직접 세팅하며 겪은 경험이 있습니다. ROS2 + GCP Vision API를 연동하거나, 로봇 데이터를 BigQuery에 저장하는 과정에서 "이걸 왜 이렇게 어렵게 해놨지?"라는 생각을 수없이 했습니다.

이 프로젝트는 기술적 배경이 있되 특정 클라우드 플랫폼 전문가는 아닌 사람이, "누구나 쉽게 따라할 수 있는 가이드"를 만들어 공유하고, 그 과정에서 지속 가능한 수익 모델을 구축하는 것입니다.
