> 최종 수정: 2025.02.07

## 1. 페르소나 개요

CloudSetup.guide의 사용자는 크게 4가지 페르소나로 분류됩니다.

```
우선순위:
🔴 P0 (핵심 타겟)  → 민지, 준호
🟡 P1 (확장 타겟)  → 서현, 제이슨
```

**기본 프로필**

| 항목 | 내용 |
|------|------|
| 이름 | 김민지 (가명) |

| 직업 | 스타트업 프로덕트 매니저 |

**배경 스토리**

민지는 15인 규모 스타트업에서 PM으로 일합니다. 최근 팀에서 "고객이 업로드한 이미지에서 텍스트를 자동 추출하는 기능"을 기획 중인데, 개발팀이 Google Cloud Vision API를 제안했습니다. 민지는 기획서를 쓰기 위해 이 API가 정확히 뭘 할 수 있는지, 무료로 얼마나 쓸 수 있는지, 어떤 제약이 있는지를 직접 확인하고 싶습니다.

**핵심 니즈**

1. "Vision API가 뭘 할 수 있는지 직접 테스트해보고 싶어"
2. "무료로 쓸 수 있는 한도가 정확히 얼마인지 알고 싶어"
3. "개발자 없이도 API 키까지는 만들어보고 싶어"
4. "이걸 기획서에 쓸 수 있는 수준으로 이해하고 싶어"

**행동 패턴**

- 검색어: "구글 비전 API 무료 사용법", "Google Vision API 가격", "이미지 텍스트 추출 API"
- 선호 콘텐츠: 스크린샷 많은 블로그 포스트, 한국어, 단계별 가이드
- 이탈 지점: 터미널/CLI 명령어가 나오면 이탈, 영어 공식 문서에서 이탈
- 재방문 트리거: "다른 API도 세팅해봐야지" (Translation API, Speech API 등)

**좌절 포인트 (Pain Points)**

- "공식 문서 들어갔는데 어디서부터 읽어야 하는지 모르겠어"
- "gcloud CLI? 터미널? 나는 그런 거 안 깔아본 적 없는데"
- "실수로 과금되면 어떡하지? 무서워서 시도를 못하겠어"
- "영어 문서밖에 없어서 번역기 돌리면서 읽느라 시간이 너무 걸려"
- "블로그 포스트 따라했는데 화면이 달라서 진행이 안 돼" (오래된 스크린샷)

**우리 사이트에서의 이상적 경험**

```
1. "구글 비전 API 사용법" 검색
2. CloudSetup.guide의 가이드 클릭
3. "아, 한국어다! 스크린샷도 있다!"
4. "먼저 GCP 계정부터 만들어야 하는구나" (선행 가이드 안내)
5. 스크린샷 따라 GCP 계정 생성 (5분)
6. 스크린샷 따라 Vision API 활성화 (3분)
7. 스크린샷 따라 API 키 생성 (5분)
8. "무료 한도: 월 1,000회" 확인 → "충분하겠다"
9. 테스트 페이지에서 이미지 업로드하여 OCR 테스트 성공
10. "이걸 기획서에 쓸 수 있겠다!"
11. 다음에 Translation API도 세팅하러 재방문
```

**핵심 인용구**

> "개발자한테 맨날 물어보기 미안해서 직접 해보고 싶은데, 어디서 시작해야 할지 모르겠어요."

| 기기 | 윈도우 노트북 |

**배경 스토리**

준호는 6개월 부트캠프를 수료하고 포트폴리오용 사이드 프로젝트를 만들고 있습니다. 음식 사진을 찍으면 칼로리를 계산해주는 앱을 만들려고 하는데, 이미지 분석을 위해 Google Vision API를 쓰고 싶습니다. 돈이 없으니 무료로 할 수 있는 범위에서 해보려고 합니다.

**핵심 니즈**

1. "무료로 API를 써서 포트폴리오 프로젝트를 만들고 싶어"
2. "GCP 계정부터 API 호출까지 빠르게 세팅하고 싶어"
3. "무료 한도를 넘기지 않도록 관리하고 싶어"
4. "나중에 다른 API도 추가할 때 참고할 곳이 필요해"
5. "이 경험을 이력서에 쓸 수 있으면 좋겠어"

**행동 패턴**

- 검색어: "GCP Vision API tutorial", "cloud run deploy python", "무료 클라우드 호스팅 추천"
- 선호 콘텐츠: 코드 예시 포함된 튜토리얼, 한/영 모두 OK, 복붙 가능한 코드
- 이탈 지점: 너무 초보적인 설명에서 이탈 ("나는 그 정도는 알아"), 유료 결제 요구 시 이탈
- 재방문 트리거: 새 프로젝트마다 다른 API 세팅 필요

**좌절 포인트 (Pain Points)**

- "GCP 콘솔이 너무 복잡해서 어디를 클릭해야 하는지 모르겠어"
- "공식 문서 예시 코드가 안 돌아가" (버전 불일치, 의존성 누락)
- "무료 한도가 정확히 어디서 확인하는 건지 모르겠어"
- "블로그 A는 이렇게 하라고 하고 블로그 B는 저렇게 하라는데 뭐가 맞아?"
- "리전을 잘못 선택하면 과금된다는데, 어느 리전을 선택해야 해?"

**우리 사이트에서의 이상적 경험**

```
1. "GCP Vision API tutorial" 검색
2. CloudSetup.guide 클릭 → 영어/한국어 선택 가능
3. 선행 가이드 확인 → 이미 GCP 계정 있으면 건너뛰기
4. Vision API 세팅 (스크린샷 + CLI 명령어 양쪽 다 제공)
5. Python 코드 예시 복붙 → 첫 API 호출 성공
6. "무료 한도 확인" 섹션에서 정확한 한도 확인
7. "활용 예시" 섹션에서 응용 아이디어 얻음
8. 다음 주에 Cloud Run 배포 가이드로 재방문
```

**핵심 인용구**

> "부트캠프에서 배운 건 로컬에서 돌리는 것까지였는데, 실제로 서비스를 배포하려니까 클라우드가 막막해요."

## 4. 페르소나별 사용자 여정 요약

```
민지 (비개발 PM):
  검색 → 한국어 가이드 발견 → 선행 가이드부터 시작 → 스크린샷 따라하기
  → API 키 생성 성공 → "와, 됐다!" → 무료 한도 확인 → 기획서에 반영
  → 다른 API도 세팅하러 재방문

준호 (주니어 개발):
  검색 → 가이드 발견 → 이미 아는 단계 건너뛰기 → API 세팅
  → 코드 예시 복붙 → 테스트 성공 → 프로젝트에 적용
  → 배포 가이드로 이동 → Cloud Run 세팅
  → 포트폴리오 완성

서현 (학생):
  검색 → 한국어 + 스크린샷 발견 → "API가 뭔지"부터 읽기
  → 무서워하며 천천히 따라하기 → "과금 안 되죠?" 확인
  → API 테스트 성공 → 스크린샷 캡처 → 과제 완성
  → "나 이런 거 해봤다!" SNS 공유

제이슨 (인디 개발):
  검색 → 비교 페이지에서 서비스 선택 → 가이드로 빠르게 세팅
  → 코드 예시 확인 → 무료 한도 계산기로 비용 확인
  → "이 규모면 무료로 충분하겠다" → 적용
  → 다음 프로젝트에서 다른 API 세팅하러 재방문
```

## 5. 콘텐츠 톤 & 보이스 가이드

### 5.1 전체 톤

**"옆에서 같이 해주는 친절한 동료"**

- 전문적이되 권위적이지 않은
- 쉽게 설명하되 무시하는 느낌이 아닌
- "이것도 모르세요?"가 아닌 "여기가 좀 헷갈릴 수 있어요"
- 기술 용어를 쓸 때는 항상 괄호로 설명 추가

### 5.2 페르소나별 톤 조절

전체 가이드는 **민지(비개발자) 수준**으로 작성하되, **준호(개발자)를 위한 추가 정보**를 별도 섹션으로 제공합니다.

```
[기본 설명 — 모든 사용자]
1. GCP 콘솔에서 좌측 메뉴의 "API 및 서비스"를 클릭하세요.
   (스크린샷)

[💡 개발자 팁 — 접힌 상태로 제공]
CLI로 하려면: gcloud services enable vision.googleapis.com
```

### 5.3 피해야 할 표현

| 피해야 할 것 | 대신 사용할 것 |
|-------------|-------------|
| "간단히" / "그냥" | "다음 단계를 따라해 주세요" |
| "당연히 아시겠지만" | (그냥 설명하기) |
| "CLI에서 실행하세요" | "웹 브라우저에서 클릭으로 할 수 있어요 (CLI 방법은 아래 참고)" |
| "Prerequisite" | "먼저 해야 할 것" |
| "인스턴스를 프로비저닝" | "서버를 만들기" |
