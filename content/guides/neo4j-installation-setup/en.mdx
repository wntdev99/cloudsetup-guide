<Step number={1} title="Install Using Docker">

### Run Neo4j with Docker

The easiest way is using Docker.

<CopyBlock language="bash" title="Run Neo4j Docker container">
docker run -d \
  --name neo4j \
  -p 7474:7474 \
  -p 7687:7687 \
  -e NEO4J_AUTH=neo4j/password123 \
  neo4j:latest
</CopyBlock>

### Using Docker Compose (Recommended)

For easier management, create a docker-compose.yml file.

<CopyBlock language="bash" title="Create docker-compose.yml file">
cat > docker-compose.yml << EOF
version: '3.8'
services:
  neo4j:
    image: neo4j:latest
    ports:
      - "7474:7474"
      - "7687:7687"
    environment:
      NEO4J_AUTH: neo4j/password123
      NEO4J_apoc_export_allow_all_databases: 'true'
    volumes:
      - neo4j_data:/var/lib/neo4j/data
      - neo4j_logs:/var/lib/neo4j/logs

volumes:
  neo4j_data:
  neo4j_logs:
EOF
</CopyBlock>

### Start with Docker Compose

<CopyBlock language="bash" title="Run Docker Compose">
docker-compose up -d
</CopyBlock>

<Step number={3} title="Run First Cypher Query">

### Create Simple Node

Enter and run this in Neo4j Browser query input field:

<CopyBlock language="text" title="First Cypher query">
CREATE (alice:Person {name: 'Alice', age: 30})
RETURN alice
</CopyBlock>

### Create Multiple Nodes and Relationships

<CopyBlock language="text" title="Create nodes and relationships">
CREATE
  (alice:Person {name: 'Alice', age: 30}),
  (bob:Person {name: 'Bob', age: 25}),
  (alice)-[:KNOWS]->(bob)
RETURN alice, bob
</CopyBlock>

### Query Data

<CopyBlock language="text" title="Query data">
MATCH (p:Person)
RETURN p.name, p.age
</CopyBlock>

<Step number={4} title="Install Python Client">

### Install Neo4j Python Driver

<CopyBlock language="bash" title="Install Neo4j driver">
pip install neo4j
</CopyBlock>

### Connect in Python

<CopyBlock language="python" title="Python connection script">
from neo4j import GraphDatabase

uri = 'neo4j://localhost:7687'
username = 'neo4j'
password = 'password123'

driver = GraphDatabase.driver(uri, auth=(username, password))

def verify_connection(tx):
  result = tx.run('RETURN 1 as value')
  return result.single()

with driver.session() as session:
  result = session.execute_read(verify_connection)
  print('Connection successful!')
</CopyBlock>

### Install Node.js Driver

<CopyBlock language="bash" title="Install Node.js driver">
npm install neo4j-driver
</CopyBlock>

### Connect in Node.js

<CopyBlock language="javascript" title="Node.js connection script">
const neo4j = require('neo4j-driver');

const uri = 'neo4j://localhost:7687';
const username = 'neo4j';
const password = 'password123';

const driver = neo4j.driver(uri, neo4j.auth.basic(username, password));

const session = driver.session();
session.run('RETURN 1')
  .then(result => {
    console.log('Connection successful!');
    session.close();
    driver.close();
  })
  .catch(error => console.error(error));
</CopyBlock>

## Neo4j Key Concepts

### Nodes

Represent entities:

<CopyBlock language="text" title="Node examples">
(alice:Person {name: 'Alice', age: 30})
(bob:Person {name: 'Bob', age: 25})
</CopyBlock>

### Relationships

Connections between nodes:

<CopyBlock language="text" title="Relationship examples">
(alice)-[:KNOWS]->(bob)
(alice)-[:WORKS_AT]->(company)
</CopyBlock>

### Labels

Types of nodes:

<CopyBlock language="text" title="Label examples">
:Person, :Company, :Movie, :User
</CopyBlock>

### Properties

Data in nodes and relationships:

<CopyBlock language="text" title="Property examples">
{name: 'Alice', age: 30, email: 'alice@example.com'}
</CopyBlock>

## Port Information

- **7474**: Neo4j Browser (web UI)
- **7687**: Bolt protocol (driver connection)

## Troubleshooting

### Connection Failed

Check if ports are in use:

<CopyBlock language="bash" title="Check ports">
lsof -i :7474
lsof -i :7687
</CopyBlock>

Use different ports:

<CopyBlock language="bash" title="Change ports">
Edit ports in docker-compose.yml
</CopyBlock>

### Authentication Failed

Reset container if you forgot password:

<CopyBlock language="bash" title="Reset container">
docker-compose down -v
docker-compose up -d
</CopyBlock>

### Performance Issues

Increase memory allocation:

<CopyBlock language="bash" title="Increase memory">
Add JAVA_OPTS environment variable in docker-compose.yml
</CopyBlock>
