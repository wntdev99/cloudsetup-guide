# Cloudflare 계정 생성 가이드

Cloudflare는 웹사이트 성능과 보안을 향상시키는 글로벌 CDN 및 보안 서비스입니다. 무료 플랜에서도 무제한 대역폭, DDoS 보호, SSL 인증서, 그리고 전 세계 300개 이상의 데이터 센터를 통한 콘텐츠 전송을 제공합니다.

  limit="Unlimited sites, global CDN, DDoS protection"
  period="forever"

/>

## 시작하기 전에

다음 항목들이 준비되어 있는지 확인하세요.

- 이메일 주소
- 도메인 이름 (보유 중인 경우)
- 도메인 등록기관 접근 권한 (DNS 설정 변경용)

<Callout type="info">
Cloudflare는 도메인 등록기관이 아닙니다. 기존 도메인의 네임서버를 Cloudflare로 변경하여 사용합니다. 도메인이 없어도 계정을 만들 수 있으며, Cloudflare Pages를 통해 무료 *.pages.dev 도메인을 받을 수 있습니다.
</Callout>

<Step number={1} title="Cloudflare 계정 생성하기">

Cloudflare 계정을 만들고 무료 플랜을 시작해봅시다.

1. [Cloudflare 회원가입 페이지](https://dash.cloudflare.com/sign-up)에 접속합니다
2. 계정 정보를 입력합니다:
   - **Email**: 본인의 이메일 주소
   - **Password**: 강력한 비밀번호 (8자 이상)
3. **Create Account** 버튼을 클릭합니다
4. 이메일로 전송된 인증 링크를 클릭하여 이메일을 인증합니다

  alt="Cloudflare 회원가입"

### 이메일 인증

1. 가입 시 입력한 이메일 계정을 확인합니다
2. Cloudflare에서 보낸 "Verify your email address" 이메일을 찾습니다
3. 이메일 내의 **Verify email** 버튼을 클릭합니다
4. 자동으로 Cloudflare 대시보드로 이동합니다

<DevTip title="Cloudflare 무료 플랜의 주요 기능">
**무료로 제공되는 기능:**
- **무제한 사이트**: 제한 없이 여러 도메인 추가 가능
- **무제한 대역폭**: 트래픽 제한 없음
- **글로벌 CDN**: 300개 이상의 데이터 센터
- **DDoS 보호**: 무료 무제한 DDoS 방어
- **무료 SSL 인증서**: 모든 사이트에 자동 HTTPS
- **Page Rules**: 3개의 페이지 규칙 (캐시, 리다이렉트 등)
- **Cloudflare Pages**: 정적 사이트 무료 호스팅 (무제한 요청)
- **Cloudflare Workers**: 10만 요청/일 무료
- **Analytics**: 기본 분석 및 트래픽 통계

**유료 플랜이 필요한 경우:**
- WAF (Web Application Firewall)
- 이미지 최적화 및 리사이징
- Page Rules 3개 이상 필요
- Load Balancing
- Argo Smart Routing (성능 향상)
- 고급 DDoS 보호 및 분석

대부분의 개인 프로젝트와 중소 규모 웹사이트는 무료 플랜으로 충분합니다.
</DevTip>

### DNS 레코드 스캔

Cloudflare가 자동으로 기존 DNS 레코드를 스캔합니다:

1. 스캔 완료까지 10-30초 정도 기다립니다
2. 발견된 DNS 레코드 목록을 확인합니다
   - **A 레코드**: IPv4 주소
   - **AAAA 레코드**: IPv6 주소
   - **CNAME 레코드**: 도메인 별칭
   - **MX 레코드**: 이메일 서버
   - **TXT 레코드**: 인증 및 설정
3. 필요시 레코드를 추가하거나 수정합니다
4. **Continue** 버튼을 클릭합니다

<Callout type="warning">
DNS 레코드가 정확하지 않으면 웹사이트나 이메일이 작동하지 않을 수 있습니다. 확실하지 않다면 자동으로 스캔된 레코드를 그대로 유지하세요.
</Callout>

<DevTip title="주요 DNS 레코드 타입 이해하기">
**A 레코드 (Address):**
- 도메인을 IPv4 주소에 연결
- 예: `example.com` → `192.0.2.1`
- 웹 서버 IP를 지정할 때 사용

**CNAME 레코드 (Canonical Name):**
- 도메인을 다른 도메인에 연결 (별칭)
- 예: `www.example.com` → `example.com`
- 서브도메인을 다른 서비스에 연결할 때 유용

**MX 레코드 (Mail Exchange):**
- 이메일 서버 주소 지정
- 예: `example.com` → `mail.google.com`
- 우선순위(priority)가 낮을수록 우선

**TXT 레코드:**
- 텍스트 정보 저장 (인증, 설정 등)
- 예: 도메인 소유권 인증, SPF, DKIM 등

**Cloudflare Proxy (주황색 구름 아이콘):**
- 켜짐 (주황색): Cloudflare CDN 및 보안 기능 활성화
- 꺼짐 (회색): DNS만 사용, 트래픽이 직접 서버로 이동
- 웹 트래픽(A, AAAA, CNAME)은 프록시 켜는 것을 권장
- 이메일(MX)이나 FTP는 프록시 끄기
</DevTip>

<Step number={3} title="네임서버 변경하기">

도메인 등록기관에서 네임서버를 Cloudflare로 변경해야 합니다.

### Cloudflare 네임서버 확인

1. Cloudflare 대시보드에 표시된 **2개의 네임서버**를 복사합니다
   - 예: `alice.ns.cloudflare.com`, `bob.ns.cloudflare.com`
2. 이 네임서버 주소를 메모장에 저장합니다

  alt="네임서버 확인"
  caption="Cloudflare 네임서버 주소를 확인합니다"
/>

### 도메인 등록기관에서 네임서버 변경

도메인을 구매한 등록기관(GoDaddy, Namecheap, 가비아, 호스팅케이알 등)에 로그인합니다:

**공통 절차:**
1. 도메인 관리 페이지로 이동합니다
2. 네임서버 또는 DNS 설정을 찾습니다
   - 예: "Nameservers", "DNS Settings", "네임서버 설정"
3. 네임서버를 **Custom** 또는 **사용자 정의**로 변경합니다
4. 기존 네임서버를 삭제하고 Cloudflare 네임서버 2개를 입력합니다
5. 변경 사항을 저장합니다

<Callout type="info">
네임서버 변경은 즉시 적용되지 않으며, 전 세계적으로 전파되는 데 **최대 24-48시간**이 걸릴 수 있습니다. 하지만 대부분 몇 시간 내에 완료됩니다.
</Callout>

**주요 등록기관별 가이드:**

**가비아 (Gabia):**
1. 가비아 로그인 → My가비아 → 도메인
2. 해당 도메인의 **관리** 클릭
3. **네임서버 설정** → **네임서버 1차, 2차 직접 설정**
4. Cloudflare 네임서버 2개 입력 → 저장

**GoDaddy:**
1. GoDaddy 로그인 → My Products → Domains
2. 도메인 옆의 **Manage DNS** 클릭
3. Nameservers 섹션에서 **Change** 클릭
4. **Use my own nameservers** 선택
5. Cloudflare 네임서버 입력 → Save

**Namecheap:**
1. Namecheap 로그인 → Domain List
2. 도메인 옆의 **Manage** 클릭
3. Nameservers → **Custom DNS** 선택
4. Cloudflare 네임서버 입력 → 체크 표시 클릭

### 네임서버 변경 확인

1. Cloudflare 대시보드로 돌아옵니다
2. **Done, check nameservers** 버튼을 클릭합니다
3. 네임서버가 변경되면 이메일로 알림을 받습니다
4. 상태가 **Active**로 변경될 때까지 기다립니다 (수 시간 ~ 24시간)

<DevTip title="네임서버 변경 확인 방법">
네임서버가 제대로 변경되었는지 확인하려면 터미널에서 다음 명령어를 실행하세요:

**Windows (PowerShell):**
```powershell
nslookup -type=NS example.com
```

**macOS/Linux:**
```bash
dig NS example.com +short
```

**출력 예시 (정상):**
```
alice.ns.cloudflare.com.
bob.ns.cloudflare.com.
```

Cloudflare 네임서버가 표시되면 성공입니다. 아직 이전 네임서버가 보인다면 전파 중이므로 조금 더 기다리세요.

**온라인 도구:**
- [DNS Checker](https://dnschecker.org/) - 전 세계 DNS 전파 상태 확인
- [What's My DNS](https://www.whatsmydns.net/) - 지역별 DNS 확인
</DevTip>

### 대시보드 개요

네임서버가 활성화되면 Cloudflare 대시보드의 모든 기능을 사용할 수 있습니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1234567890/guides/cloudflare-account-setup/step4-dashboard-ko.png"
  alt="Cloudflare 대시보드"
  caption="Cloudflare 대시보드의 주요 메뉴를 확인합니다"
/>

### 주요 메뉴 소개

**1. Analytics (분석)**
- 트래픽 통계 및 요청 수
- 대역폭 사용량
- 위협 차단 통계
- 방문자 지역 분포

**2. DNS**
- DNS 레코드 추가/수정/삭제
- A, AAAA, CNAME, MX, TXT 레코드 관리
- 프록시 상태 (주황색/회색 구름) 토글
- DNSSEC 설정

**3. SSL/TLS**
- SSL 인증서 자동 발급 (무료)
- SSL 모드 선택:
  - **Flexible**: Cloudflare ↔ 방문자 간 암호화 (서버는 HTTP)
  - **Full**: 전체 암호화 (자체 서명 인증서 허용)
  - **Full (strict)**: 전체 암호화 + 유효한 인증서 필요
- Always Use HTTPS 설정

**4. Firewall (방화벽)**
- IP 차단/허용 규칙
- User Agent 차단
- 국가별 차단
- Rate Limiting (유료)

**5. Speed (속도 최적화)**
- Auto Minify: HTML, CSS, JS 압축
- Brotli 압축 활성화
- HTTP/3 (QUIC) 지원
- Rocket Loader: JavaScript 비동기 로딩

**6. Caching (캐싱)**
- 캐시 레벨 설정
- 브라우저 캐시 TTL
- 캐시 삭제 (Purge Cache)
- Page Rules: 특정 경로의 캐시 규칙

**7. Workers**
- Edge에서 실행되는 서버리스 함수
- 무료: 10만 요청/일
- JavaScript로 작성

**8. Pages**
- 정적 사이트 배포 (Git 연동)
- 무료: 무제한 요청, 500 빌드/월
- Next.js, React, Vue 등 지원

### 초기 권장 설정

빠르게 성능과 보안을 향상시키려면 다음 설정을 활성화하세요:

**1. SSL/TLS → Overview → Full (strict) 선택**
- 서버에 유효한 SSL 인증서가 있는 경우

**2. SSL/TLS → Edge Certificates → Always Use HTTPS → On**
- HTTP 요청을 자동으로 HTTPS로 리다이렉트

**3. Speed → Optimization → Auto Minify → HTML, CSS, JS 체크**
- 파일 크기 자동 압축

**4. Speed → Optimization → Brotli → On**
- 더 나은 압축 알고리즘 사용

**5. Caching → Configuration → Caching Level → Standard**
- 기본 캐싱 활성화

<DevTip title="Cloudflare 활용 팁">
**성능 향상 체크리스트:**
- ✅ Auto Minify 활성화 (HTML, CSS, JS)
- ✅ Brotli 압축 활성화
- ✅ HTTP/3 활성화
- ✅ DNS 레코드에 프록시(주황색 구름) 활성화
- ✅ Always Use HTTPS 활성화

**보안 강화 체크리스트:**
- ✅ SSL/TLS를 Full (strict)로 설정
- ✅ Firewall에서 의심스러운 IP 차단
- ✅ Security Level을 Medium 이상으로 설정
- ✅ Browser Integrity Check 활성화

**주의사항:**
- Development Mode를 켜면 캐싱이 3시간 동안 비활성화됩니다 (개발 중에만 사용)
- Page Rules는 무료 플랜에서 3개까지만 생성 가능
- Rocket Loader는 일부 JavaScript와 충돌할 수 있으므로 필요시 비활성화
- Cloudflare 프록시를 사용하면 실제 서버 IP가 숨겨집니다 (보안 장점)
</DevTip>

## 문제 해결

### 네임서버가 24시간이 지나도 활성화되지 않아요

**확인 사항:**
1. 도메인 등록기관에서 네임서버를 정확히 입력했는지 확인
2. 네임서버 2개를 모두 입력했는지 확인
3. 도메인이 잠금(Lock) 상태가 아닌지 확인

**해결 방법:**
- 도메인 등록기관에 문의하여 네임서버 변경 상태 확인
- `nslookup -type=NS your-domain.com` 명령어로 현재 네임서버 확인

### SSL 인증서 오류가 발생해요

**원인:**
- SSL 모드가 잘못 설정됨
- 서버에 유효한 SSL 인증서가 없음

**해결 방법:**
1. SSL/TLS → Overview에서 SSL 모드 변경
2. 서버에 SSL 인증서가 없는 경우 → **Flexible** 선택
3. 서버에 유효한 인증서가 있는 경우 → **Full (strict)** 선택

### 웹사이트가 작동하지 않아요 (DNS_PROBE_FINISHED_NXDOMAIN)

**원인:**
- DNS 레코드가 올바르지 않음
- A 레코드가 없거나 IP 주소가 잘못됨

**해결 방법:**
1. DNS 메뉴로 이동
2. A 레코드 확인 (@ 또는 www)
3. 올바른 서버 IP 주소를 입력했는지 확인
4. 프록시 상태(주황색 구름)가 활성화되어 있는지 확인

### 이메일을 받을 수 없어요

**원인:**
- MX 레코드에 프록시가 활성화됨

**해결 방법:**
1. DNS 메뉴로 이동
2. MX 레코드의 프록시를 **비활성화** (회색 구름)
3. MX 레코드는 프록시를 사용하면 안 됩니다
