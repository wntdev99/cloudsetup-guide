# Cloudflare Account Setup Guide

Cloudflare is a global CDN and security service that improves website performance and security. Even the free plan provides unlimited bandwidth, DDoS protection, SSL certificates, and content delivery through over 300 data centers worldwide.

## Before You Begin

Make sure you have the following ready.

- Email address
- Domain name (if you own one)
- Domain registrar access (for DNS configuration changes)

<Callout type="info">
Cloudflare is not a domain registrar. You change your existing domain's nameservers to Cloudflare. You can create an account even without a domain, and get a free *.pages.dev domain through Cloudflare Pages.
</Callout>

<Step number={1} title="Create Cloudflare Account">

Let's create a Cloudflare account and start with the free plan.

1. Visit the [Cloudflare sign-up page](https://dash.cloudflare.com/sign-up)
2. Enter your account information:
   - **Email**: Your email address
   - **Password**: Strong password (8+ characters)
3. Click the **Create Account** button
4. Click the verification link sent to your email to verify your address

### Email Verification

1. Check the email account you used during signup
2. Find the "Verify your email address" email from Cloudflare
3. Click the **Verify email** button in the email
4. You'll be automatically redirected to the Cloudflare dashboard

<DevTip title="Cloudflare Free Plan Key Features">
**Features included for free:**
- **Unlimited sites**: Add multiple domains without limits
- **Unlimited bandwidth**: No traffic restrictions
- **Global CDN**: 300+ data centers worldwide

Most personal projects and small-to-medium websites are well-served by the free plan.

<Step number={2} title="Add Your Site (Domain)">

Let's add your domain to Cloudflare.

1. Click the **Add a Site** button in the Cloudflare dashboard
2. Enter your domain name
   - Example: `example.com` (root domain only, no subdomain)
3. Click the **Add site** button
4. Select a plan: Choose **Free** then click **Continue**

### DNS Records Scan

Cloudflare will automatically scan your existing DNS records:

1. Wait 10-30 seconds for the scan to complete
2. Review the list of discovered DNS records
   - **A record**: IPv4 address
   - **AAAA record**: IPv6 address
   - **CNAME record**: Domain alias
   - **MX record**: Email server
   - **TXT record**: Verification and configuration
3. Add or modify records if needed
4. Click the **Continue** button

<Callout type="warning">
If DNS records are incorrect, your website or email may not work. If unsure, keep the automatically scanned records as-is.
</Callout>

<DevTip title="Understanding Main DNS Record Types">
**A Record (Address):**
- Links domain to IPv4 address
- Example: `example.com` → `192.0.2.1`
- Used to specify web server IP

**CNAME Record (Canonical Name):**
- Links domain to another domain (alias)
- Example: `www.example.com` → `example.com`
- Useful for connecting subdomains to other services

**MX Record (Mail Exchange):**
- Specifies email server address
- Example: `example.com` → `mail.google.com`
- Lower priority number = higher priority

**TXT Record:**
- Stores text information (verification, configuration, etc.)
- Example: Domain ownership verification, SPF, DKIM, etc.

**Cloudflare Proxy (orange cloud icon):**
- On (orange): Cloudflare CDN and security features enabled
- Off (gray): DNS only, traffic goes directly to server
- Recommended to enable proxy for web traffic (A, AAAA, CNAME)
- Turn off proxy for email (MX) or FTP
</DevTip>

<Step number={3} title="Change Nameservers">

You need to change nameservers to Cloudflare at your domain registrar.

### Check Cloudflare Nameservers

1. Copy the **2 nameservers** shown in the Cloudflare dashboard
   - Example: `alice.ns.cloudflare.com`, `bob.ns.cloudflare.com`
2. Save these nameserver addresses in a notepad

### Change Nameservers at Domain Registrar

Log in to the registrar where you purchased your domain (GoDaddy, Namecheap, Google Domains, etc.):

**Common procedure:**
1. Navigate to domain management page
2. Find nameserver or DNS settings
   - Example: "Nameservers", "DNS Settings", "Name Server Configuration"
3. Change nameservers to **Custom** or **Use custom nameservers**
4. Delete existing nameservers and enter the 2 Cloudflare nameservers
5. Save changes

<Callout type="info">
Nameserver changes don't take effect immediately. Global propagation can take **up to 24-48 hours**, though most complete within a few hours.
</Callout>

**Guides for major registrars:**

**GoDaddy:**
1. GoDaddy login → My Products → Domains
2. Click **Manage DNS** next to domain
3. Click **Change** in Nameservers section
4. Select **Use my own nameservers**
5. Enter Cloudflare nameservers → Save

**Namecheap:**
1. Namecheap login → Domain List
2. Click **Manage** next to domain
3. Nameservers → Select **Custom DNS**
4. Enter Cloudflare nameservers → Click checkmark

**Google Domains:**
1. Google Domains login → My domains
2. Click domain name
3. DNS → Name servers → **Use custom name servers**
4. Enter Cloudflare nameservers → Save

### Verify Nameserver Change

1. Return to Cloudflare dashboard
2. Click **Done, check nameservers** button
3. You'll receive an email notification when nameservers are changed
4. Wait for status to change to **Active** (few hours ~ 24 hours)

<DevTip title="How to Verify Nameserver Change">
To check if nameservers were properly changed, run this command in terminal:

**Windows (PowerShell):**
```powershell
nslookup -type=NS example.com
```

**macOS/Linux:**
```bash
dig NS example.com +short
```

**Example output (successful):**
```
alice.ns.cloudflare.com.
bob.ns.cloudflare.com.
```

If Cloudflare nameservers appear, it's successful. If old nameservers still show, propagation is ongoing - wait a bit longer.

**Online tools:**
- [DNS Checker](https://dnschecker.org/) - Check DNS propagation worldwide
- [What's My DNS](https://www.whatsmydns.net/) - Check DNS by region
</DevTip>

**3. SSL/TLS**
- Automatic SSL certificate issuance (free)
- SSL mode selection:
  - **Flexible**: Cloudflare ↔ visitor encryption (server uses HTTP)
  - **Full**: Full encryption (allows self-signed certificates)
  - **Full (strict)**: Full encryption + valid certificate required
- Always Use HTTPS setting

**4. Firewall**
- IP block/allow rules
- User Agent blocking
- Country-based blocking
- Rate Limiting (paid)

**5. Speed (Optimization)**
- Auto Minify: HTML, CSS, JS compression
- Brotli compression activation
- HTTP/3 (QUIC) support
- Rocket Loader: Asynchronous JavaScript loading

**6. Caching**
- Cache level settings
- Browser cache TTL
- Purge cache
- Page Rules: Cache rules for specific paths

**7. Workers**
- Serverless functions running at the edge
- Free: 100,000 requests/day
- Written in JavaScript

**8. Pages**
- Static site deployment (Git integration)
- Free: Unlimited requests, 500 builds/month
- Supports Next.js, React, Vue, etc.

### Recommended Initial Settings

To quickly improve performance and security, enable these settings:

**1. SSL/TLS → Overview → Select Full (strict)**
- If server has valid SSL certificate

**2. SSL/TLS → Edge Certificates → Always Use HTTPS → On**
- Automatically redirect HTTP requests to HTTPS

**3. Speed → Optimization → Auto Minify → Check HTML, CSS, JS**
- Automatic file size compression

**4. Speed → Optimization → Brotli → On**
- Use better compression algorithm

**5. Caching → Configuration → Caching Level → Standard**
- Enable basic caching

<DevTip title="Cloudflare Usage Tips">
**Performance improvement checklist:**
- ✅ Enable Auto Minify (HTML, CSS, JS)
- ✅ Enable Brotli compression
- ✅ Enable HTTP/3
- ✅ Enable proxy (orange cloud) on DNS records
- ✅ Enable Always Use HTTPS

**Security enhancement checklist:**
- ✅ Set SSL/TLS to Full (strict)
- ✅ Block suspicious IPs in Firewall
- ✅ Set Security Level to Medium or higher
- ✅ Enable Browser Integrity Check

**Cautions:**
- Enabling Development Mode disables caching for 3 hours (use only during development)
- Free plan allows only 3 Page Rules
- Rocket Loader may conflict with some JavaScript - disable if needed
- Using Cloudflare proxy hides your actual server IP (security benefit)
</DevTip>

## Troubleshooting

### Nameservers not active after 24 hours

**Check:**
1. Verify nameservers were entered correctly at domain registrar
2. Confirm both nameservers were entered
3. Check domain is not in lock status

**Solution:**
- Contact domain registrar to verify nameserver change status
- Check current nameservers with `nslookup -type=NS your-domain.com`

### SSL certificate error

**Cause:**
- SSL mode incorrectly configured
- Server doesn't have valid SSL certificate

**Solution:**
1. Change SSL mode in SSL/TLS → Overview
2. If server has no SSL certificate → Select **Flexible**
3. If server has valid certificate → Select **Full (strict)**

### Website not working (DNS_PROBE_FINISHED_NXDOMAIN)

**Cause:**
- DNS records incorrect
- A record missing or IP address wrong

**Solution:**
1. Go to DNS menu
2. Check A record (@ or www)
3. Verify correct server IP address entered
4. Check proxy status (orange cloud) is enabled

### Cannot receive email

**Cause:**
- Proxy enabled on MX record

**Solution:**
1. Go to DNS menu
2. **Disable** proxy on MX record (gray cloud)
3. MX records should not use proxy
