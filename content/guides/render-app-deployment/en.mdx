<Step number={1} title="Prepare GitHub Repository">

Prepare your repository before deployment.

**Required files:**

- **package.json** (Node.js) or **requirements.txt** (Python) or **go.mod** (Go)
- **Procfile** (optional - Render config)

**Procfile example (Node.js):**

<CopyBlock
  code="web: npm start"
  language="bash"
/>

**Procfile example (Python):**

<CopyBlock
  code="web: gunicorn app:app"
  language="bash"
/>

**Check package.json start script (Node.js):**

<CopyBlock
  code="\"scripts\": {
  \"start\": \"node server.js\"
}"
  language="json"
/>

**Port configuration (critical):**

Render uses PORT environment variable. Your app must listen on it:

<CopyBlock
  code="const port = process.env.PORT || 3000;
app.listen(port, () => {
  console.log('Server is running on port ' + port);
});"
  language="javascript"
/>

Or Python:

<CopyBlock
  code="import os
port = int(os.environ.get('PORT', 5000))
app.run(host='0.0.0.0', port=port)"
  language="python"
/>

<Step number={2} title="Create and Deploy Web Service in Render">

In Render dashboard:

1. Click **New +** button.
2. Select **Web Service**.
3. Select your GitHub repository.
4. Verify settings:
   - **Name**: App name
   - **Runtime**: Node, Python, Go, etc.
   - **Build Command**: npm install, pip install, etc.
   - **Start Command**: npm start, python app.py, etc.

5. Select **Free** instance (90-day free).
6. Click **Deploy**.

Watch deployment progress in logs.

<Step number={3} title="Set Environment Variables">

Configure database URLs, API keys, etc. as environment variables.

**Add environment variables in Render:**

1. Click your deployed app in dashboard.
2. Go to **Environment** tab.
3. Click **Add Environment Variable**.
4. Enter **Key** (e.g., DATABASE_URL).
5. Enter **Value** (database connection string).
6. Click **Save**.

**Add multiple variables:**

<CopyBlock
  code="DATABASE_URL=postgresql://...
API_KEY=sk_test_...
REDIS_URL=redis://..."
  language="bash"
/>

**Redeploy after changes:**

Auto-redeploy or click **Manual Deploy** button.

<Step number={4} title="Deployment Complete and Monitoring">

When deployment finishes:

1. Check **URL** (e.g., https://my-app.onrender.com).
2. Visit URL in browser.
3. Verify app works correctly.

**Check logs:**

1. Go to **Logs** tab for real-time logs.
2. Find errors in logs if issues occur.

**GitHub auto-deploy:**

1. Enable **Auto-Deploy** toggle (default).
2. Select **main branch**.
3. GitHub push triggers auto-deploy.

**Connect domain:**

1. Go to **Settings** tab.
2. Add **Custom Domain**.
3. Configure DNS records.

**Restart/delete app:**

1. **Settings > Restart Service** - Restart app.
2. **Settings > Delete Service** - Delete app.

## Frequently Asked Questions

<Callout type="info">
**Q. I get 502 error after deployment**

A. Check:
- Are you using PORT environment variable?
- Is start command correct?
- Did dependencies install?
- Check logs for errors
</Callout>

<Callout type="info">
**Q. Web service auto-stops**

A. 90-day trial ended or free instance auto-paused (cost savings). Upgrade to paid or manually restart.
</Callout>

<Callout type="info">
**Q. I have a domain. Can I connect it?**

A. Yes! In **Custom Domain** settings, point DNS record as CNAME.
</Callout>

## Troubleshooting

### Deployment fails

- Check build logs: **Logs** tab
- Verify buildCommand and startCommand
- Check dependency files (package.json, requirements.txt)

### App won't run

- Check logs
- Verify PORT environment variable
- Check port binds to 0.0.0.0

### Auto-deploy not working

- Verify **Auto-Deploy** is enabled in dashboard
- Check GitHub Webhook
- Test with manual deploy
