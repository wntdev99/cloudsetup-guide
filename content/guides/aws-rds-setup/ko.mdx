# AWS RDS 설정하기

AWS RDS(Relational Database Service)는 클라우드에서 관리형 관계형 데이터베이스를 제공하는 서비스입니다. 이 가이드에서는 MySQL 또는 PostgreSQL 데이터베이스를 생성하고 연결하는 방법을 안내합니다.

## 사전 준비사항

- AWS 계정 ([aws-account-setup](/guides/aws-account-setup) 참고)
- IAM 사용자 및 액세스 키 ([aws-iam-user-create](/guides/aws-iam-user-create) 참고)
- 데이터베이스 클라이언트 (MySQL Workbench, pgAdmin, DBeaver 등)

<Step number={1} title="RDS 대시보드 접속 및 데이터베이스 생성 시작">

AWS 콘솔에서 RDS 서비스로 이동하여 데이터베이스 생성을 시작합니다.

1. [AWS Console](https://console.aws.amazon.com/)에 로그인합니다
2. 상단 검색창에 "RDS"를 입력하고 선택합니다
3. **데이터베이스 생성** 버튼을 클릭합니다

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/guides/aws-rds-setup/step1-create-database-ko.png"
  alt="RDS 데이터베이스 생성 버튼"
  caption="데이터베이스 생성 버튼을 클릭하여 새 DB 인스턴스 생성을 시작합니다"
/>

### 데이터베이스 생성 방법 선택

4. **데이터베이스 생성 방식 선택**에서 **표준 생성** 선택
5. **엔진 옵션**에서 다음 중 하나를 선택합니다:
   - **MySQL** (권장 - 가장 널리 사용됨)
   - **PostgreSQL** (고급 기능 필요 시)
   - **MariaDB** (MySQL 호환)

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/guides/aws-rds-setup/step1-engine-selection-ko.png"
  alt="데이터베이스 엔진 선택"
  caption="MySQL 또는 PostgreSQL 엔진을 선택합니다"
/>

6. **엔진 버전**: 최신 버전 선택 (예: MySQL 8.0.35)

<Callout type="info">
MySQL과 PostgreSQL 모두 프리 티어를 지원합니다. MySQL이 더 널리 사용되며, PostgreSQL은 고급 SQL 기능이 필요할 때 적합합니다.
</Callout>

</Step>

<Step number={2} title="템플릿 및 인스턴스 설정">

프리 티어 템플릿을 선택하고 데이터베이스 인스턴스를 구성합니다.

### 템플릿 선택

1. **템플릿** 섹션에서 **프리 티어** 선택
   - 이렇게 하면 프리 티어 적용 가능한 옵션만 표시됩니다

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/guides/aws-rds-setup/step2-free-tier-template-ko.png"
  alt="프리 티어 템플릿 선택"
  caption="프리 티어 템플릿을 선택하여 무료 사용 가능한 옵션으로 제한합니다"
/>

### 설정 구성

2. **설정** 섹션에서:
   - **DB 인스턴스 식별자**: `my-database` (고유한 이름 입력)
   - **마스터 사용자 이름**: `admin` (기본값 또는 원하는 이름)
   - **마스터 암호**: 강력한 암호 입력 (최소 8자)
   - **암호 확인**: 동일한 암호 재입력

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/guides/aws-rds-setup/step2-credentials-ko.png"
  alt="DB 자격 증명 설정"
  caption="데이터베이스 마스터 사용자 이름과 암호를 설정합니다"
/>

<Callout type="warning">
마스터 암호는 안전하게 보관하세요. 이 암호는 데이터베이스 접속에 필요하며, 분실 시 복구가 어렵습니다.
</Callout>

### 인스턴스 구성

3. **인스턴스 구성** 섹션:
   - **DB 인스턴스 클래스**: **db.t2.micro** 또는 **db.t3.micro** (프리 티어)
   - 버스터블 클래스 (t 클래스 포함) 선택됨

4. **스토리지** 섹션:
   - **스토리지 유형**: **범용 SSD (gp2)** 또는 **gp3**
   - **할당된 스토리지**: **20 GiB** (프리 티어 최대치)
   - **스토리지 자동 조정**: 체크 해제 (비용 관리를 위해)

<Step number={3} title="연결 및 보안 그룹 설정">

데이터베이스 연결 설정과 보안 그룹을 구성합니다.

### 연결 설정

1. **연결** 섹션에서:
   - **컴퓨팅 리소스**: **EC2 컴퓨팅 리소스에 연결 안 함** 선택 (독립 실행형)
   - **네트워크 유형**: **IPv4**
   - **Virtual Private Cloud (VPC)**: 기본 VPC 선택
   - **퍼블릭 액세스**: **예** 선택 (외부에서 접속하려면)

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/guides/aws-rds-setup/step3-connectivity-ko.png"
  alt="연결 설정"
  caption="퍼블릭 액세스를 활성화하여 외부에서 데이터베이스에 접속할 수 있도록 합니다"
/>

<Callout type="info">
**퍼블릭 액세스 옵션:**
- **예**: 인터넷에서 접속 가능 (보안 그룹으로 접근 제어)
- **아니오**: VPC 내부에서만 접속 가능 (더 안전)

개발/테스트 목적이라면 "예"를 선택하고, 프로덕션 환경에서는 VPC 내부 접속을 권장합니다.
</Callout>

### VPC 보안 그룹

2. **VPC 보안 그룹** 섹션:
   - **기존 항목 선택** 또는 **새로 생성** 선택
   - 새로 생성하는 경우 이름: `rds-mysql-sg`

3. **가용 영역**: **기본 설정 없음** (자동 선택)

### 데이터베이스 인증

4. **데이터베이스 인증** 섹션:
   - **암호 인증** 선택 (기본값)

</Step>

<Step number={4} title="추가 구성 및 데이터베이스 생성">

백업, 모니터링 등 추가 설정을 구성하고 데이터베이스를 생성합니다.

### 추가 구성

1. **추가 구성** 섹션을 펼칩니다

2. **데이터베이스 옵션**:
   - **초기 데이터베이스 이름**: `mydb` (선택사항이지만 권장)
   - **DB 파라미터 그룹**: 기본값 사용
   - **옵션 그룹**: 기본값 사용

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/guides/aws-rds-setup/step4-additional-config-ko.png"
  alt="추가 구성"
  caption="초기 데이터베이스 이름과 백업 설정을 구성합니다"
/>

3. **백업**:
   - **자동 백업 활성화**: 체크 (권장)
   - **백업 보존 기간**: **7일** (프리 티어 범위 내)
   - **백업 기간**: 원하는 시간대 선택

4. **암호화**:
   - 프리 티어에서는 기본 설정 사용

5. **모니터링**:
   - **Enhanced Monitoring 활성화**: 체크 해제 (추가 비용 발생)

6. **로그 내보내기**: 필요한 경우 선택 (CloudWatch Logs로 전송)

### 유지 관리

7. **유지 관리** 섹션:
   - **자동 마이너 버전 업그레이드 활성화**: 체크 (보안 패치 자동 적용)

### 삭제 방지

8. **삭제 방지**: 프로덕션 환경에서는 체크, 테스트 환경에서는 체크 해제

### 데이터베이스 생성

9. 오른쪽 **월별 추정 요금**에서 프리 티어 범위 내인지 확인합니다
10. **데이터베이스 생성** 버튼을 클릭합니다

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/guides/aws-rds-setup/step4-create-ko.png"
  alt="데이터베이스 생성"
  caption="모든 설정을 확인하고 데이터베이스를 생성합니다"
/>

11. 데이터베이스 생성이 시작되며, **사용 가능** 상태가 되기까지 5~10분 소요됩니다

<Callout type="info">
데이터베이스가 생성되는 동안 상태가 "생성 중"에서 "백업 중"을 거쳐 "사용 가능"으로 변경됩니다.
</Callout>

</Step>

<Step number={5} title="데이터베이스 연결 및 테스트">

생성된 데이터베이스에 연결하고 정상 작동을 확인합니다.

### 엔드포인트 확인

1. RDS 대시보드에서 생성한 데이터베이스를 선택합니다
2. **연결 & 보안** 탭에서 다음 정보를 확인합니다:
   - **엔드포인트**: `my-database.xxxxxxxxxx.ap-northeast-2.rds.amazonaws.com`
   - **포트**: `3306` (MySQL) 또는 `5432` (PostgreSQL)

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/guides/aws-rds-setup/step5-endpoint-ko.png"
  alt="DB 엔드포인트 확인"
  caption="데이터베이스 연결에 필요한 엔드포인트와 포트 정보를 확인합니다"
/>

### 보안 그룹 인바운드 규칙 수정

3. **VPC 보안 그룹**에서 보안 그룹 이름을 클릭합니다
4. **인바운드 규칙** 탭에서 **인바운드 규칙 편집** 클릭
5. 다음 규칙을 추가합니다:

**MySQL의 경우:**
- 유형: **MYSQL/Aurora**
- 포트: **3306**
- 소스: **내 IP** (또는 특정 IP 범위)

**PostgreSQL의 경우:**
- 유형: **PostgreSQL**
- 포트: **5432**
- 소스: **내 IP** (또는 특정 IP 범위)

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/guides/aws-rds-setup/step5-security-group-ko.png"
  alt="보안 그룹 인바운드 규칙"
  caption="데이터베이스 포트에 대한 인바운드 규칙을 추가합니다"
/>

6. **규칙 저장** 버튼을 클릭합니다

<Callout type="warning">
보안을 위해 소스를 "내 IP" 또는 특정 IP 범위로 제한하세요. 모든 IP(0.0.0.0/0)를 허용하면 보안 위험이 증가합니다.
</Callout>

### MySQL 연결 테스트

7. MySQL 클라이언트를 사용하여 연결합니다:

<CopyBlock language="bash">
mysql -h my-database.xxxxxxxxxx.ap-northeast-2.rds.amazonaws.com \
  -P 3306 \
  -u admin \
  -p
</CopyBlock>

또는 연결 문자열:

<CopyBlock language="text">
mysql://admin:password@my-database.xxxxxxxxxx.ap-northeast-2.rds.amazonaws.com:3306/mydb
</CopyBlock>

### PostgreSQL 연결 테스트

8. PostgreSQL 클라이언트를 사용하여 연결합니다:

<CopyBlock language="bash">
psql -h my-database.xxxxxxxxxx.ap-northeast-2.rds.amazonaws.com \
  -p 5432 \
  -U admin \
  -d mydb
</CopyBlock>

또는 연결 문자열:

<CopyBlock language="text">
postgresql://admin:password@my-database.xxxxxxxxxx.ap-northeast-2.rds.amazonaws.com:5432/mydb
</CopyBlock>

<CopyBlock language="javascript">

  port: 3306,
  user: 'admin',
  password: 'your-password',
  database: 'mydb'

connection.connect();
</CopyBlock>

<CopyBlock language="python">

conn = psycopg2.connect(

    port=5432,
    user="admin",
    password="your-password",
    database="mydb"

</CopyBlock>

## 데이터베이스 관리 팁

### 백업 및 복원

- **자동 백업**: 매일 자동으로 수행 (설정한 백업 기간 동안)
- **스냅샷**: 수동 스냅샷으로 특정 시점 백업
- **복원**: 자동 백업 또는 스냅샷에서 새 DB 인스턴스로 복원

### 성능 모니터링

1. RDS 대시보드에서 **모니터링** 탭 확인
2. CPU 사용률, 연결 수, IOPS 등 메트릭 확인
3. CloudWatch에서 더 상세한 메트릭 조회

### 비용 최적화

- **인스턴스 중지**: 일시적으로 사용하지 않을 때 (최대 7일)
- **스토리지 조정**: 필요한 만큼만 할당
- **백업 보존 기간**: 7일 이상은 추가 비용 발생

## 문제 해결

### 데이터베이스에 연결할 수 없습니다

1. **보안 그룹 확인**: 해당 포트가 열려 있는지 확인
2. **퍼블릭 액세스**: "예"로 설정되어 있는지 확인
3. **엔드포인트**: 올바른 엔드포인트를 사용하고 있는지 확인
4. **자격 증명**: 사용자 이름과 암호가 정확한지 확인

### 프리 티어 초과 걱정

1. **인스턴스 클래스**: db.t2.micro 또는 db.t3.micro 사용 확인
2. **스토리지**: 20GB 이하 유지
3. **백업**: 20GB 이하 백업 스토리지 유지
4. **Billing 알람**: CloudWatch에서 비용 알람 설정

### 성능이 느립니다

1. **인스턴스 크기**: 프리 티어는 제한적인 성능 제공
2. **연결 수**: 동시 연결 수가 많으면 성능 저하
3. **쿼리 최적화**: 느린 쿼리 로그 확인 및 인덱스 추가
4. **스토리지 IOPS**: gp3로 업그레이드하여 성능 개선

<Callout type="tip">
개발 환경에서는 RDS 대신 로컬 데이터베이스를 사용하여 프리 티어 시간을 절약할 수 있습니다. 프로덕션 테스트 시에만 RDS를 사용하세요.
</Callout>
