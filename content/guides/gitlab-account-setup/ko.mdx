# GitLab 계정 만들고 프로젝트 설정하기

GitLab은 **Git 저장소 호스팅 및 DevOps 플랫폼**입니다. GitHub처럼 코드를 저장하고 관리할 수 있으며, 내장된 CI/CD 파이프라인으로 자동화된 배포를 구현할 수 있습니다. 무료로 무제한의 비공개 저장소를 사용할 수 있습니다.

<Step number={1} title="GitLab 가입">

### GitLab 웹사이트 방문

1. [GitLab.com](https://gitlab.com)을 방문합니다
2. **가입** 또는 **Register** 버튼을 클릭합니다
3. 가입 방법을 선택합니다:
   - 이메일 주소
   - GitHub 계정
   - Google 계정
   - Microsoft 계정

### 이메일로 가입

<CopyBlock language="text" title="가입 정보 입력">

이메일 주소: john@example.com
비밀번호: 안전한 비밀번호 입력
</CopyBlock>

가입 버튼을 클릭하면 확인 이메일이 발송됩니다.

### 확인 이메일 클릭

1. 이메일 받은편지함 확인

### 프로필 정보 입력

1. **이름**: 실명 또는 표시 이름
2. **사용자명**: 고유한 사용자명 (이미 선택됨)
3. **회사** (선택사항): 회사명
4. **웹사이트** (선택사항): 포트폴리오나 블로그 URL

<Step number={3} title="첫 번째 프로젝트 생성">

### 새 프로젝트 생성

1. GitLab 대시보드에서 **프로젝트 생성** 또는 **New Project** 클릭
2. **새로 만들기[Create blank project]** 선택

### 프로젝트 정보 입력

<CopyBlock language="text" title="프로젝트 설정">
프로젝트 이름: my-first-app
설명: My first GitLab project
가시성: Private (비공개)
Initialize with README: 체크
</CopyBlock>

옵션 설명:
- **프로젝트 이름**: 프로젝트 URL의 일부가 됩니다
- **설명**: 프로젝트 소개
- **가시성**: Public (공개) 또는 Private (비공개)
  - Private: 초대한 사람만 볼 수 있음
  - Public: 누구나 볼 수 있음
- **README 초기화**: 기본 README.md 파일 생성

<Step number={4} title="SSH 키 설정 및 저장소 복제">

### SSH 키 생성

터미널에서 SSH 키를 생성합니다:

<CopyBlock language="bash" title="SSH 키 생성">
ssh-keygen -t ed25519 -C "john@example.com"
</CopyBlock>

프롬프트가 나타나면:
- **파일 위치**: Enter 누르면 기본값 `~/.ssh/id_ed25519` 사용
- **패스프레이즈**: 비어있어도 되지만 보안을 위해 입력 권장

### 공개 키 GitLab에 추가

<CopyBlock language="bash" title="공개 키 내용 확인">
cat ~/.ssh/id_ed25519.pub
</CopyBlock>

출력된 내용을 복사합니다.

### GitLab SSH 키 등록

1. GitLab에서 **설정[Settings]** > **SSH 키[SSH Keys]** 선택
2. **새 SSH 키 추가** 클릭
3. 공개 키 내용 붙여넣기
4. **제목** 입력 (예: "My Laptop")
5. **키 추가** 클릭

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/gitlab/step4-ssh-setup-ko.png"
  alt="SSH 키 설정"
  caption="SSH 키를 생성하고 GitLab에 등록합니다"
/>

### 저장소 복제

<CopyBlock language="bash" title="저장소 복제">
git clone git@gitlab.com:username/my-first-app.git
cd my-first-app
</CopyBlock>

## 팀 관리

### 팀원 초대

1. **프로젝트 설정[Project Settings]** > **멤버[Members]**
2. **멤버 초대[Invite members]** 클릭
3. 이메일 주소 입력
4. 역할 선택:
   - **Guest** - 읽기 전용
   - **Reporter** - 이슈 보고 및 코멘트
   - **Developer** - 코드 푸시 및 수정
   - **Maintainer** - 전체 권한

### 그룹 생성

여러 프로젝트를 함께 관리하려면:

1. GitLab 메인 페이지에서 **그룹 생성[New Group]**
2. 그룹 이름과 슬러그 입력
3. 그룹 내에서 프로젝트 생성 및 관리

## 문제 해결

### SSH 키 연결 오류

<CopyBlock language="bash" title="SSH 연결 테스트">
ssh -T git@gitlab.com
</CopyBlock>

응답:
```
Welcome to GitLab, @username!
```

### HTTPS 대신 SSH 사용

저장소 클론 시 HTTPS 대신 SSH를 사용하려면:

<CopyBlock language="bash" title="저장소 리모트 URL 변경">
git remote set-url origin git@gitlab.com:username/my-first-app.git
</CopyBlock>

### 저장소가 보이지 않습니다

1. 올바른 계정으로 로그인 확인
2. 프로젝트 가시성 확인 (Private/Public)
3. 초대 수락 여부 확인
