# Create PlanetScale Account and Setup Database

PlanetScale is a cloud database service based on MySQL. Using Vitess technology, it provides a highly scalable database. Let's create a PlanetScale account and your first database!

## What is PlanetScale?

<Callout type="info">
üöÄ **PlanetScale is a MySQL-compatible database service**

Key features:
- **MySQL compatible**: Use existing MySQL tools and libraries
- **Vitess powered**: Auto-sharding for unlimited scaling
- **Branching**: Create database versions like Git branches
- **Auto backup**: 30-day retention
- **Online migrations**: Zero-downtime schema changes

Examples:
- `myapp_db`: Production database
- `development`: Development environment branch
- `staging`: Staging environment branch
</Callout>

<Step number={2} title="Email Verification and Profile Setup">

### Email Verification

1. Verification link sent to registered email
2. Click **Verify email** link in the email
3. Dashboard appears after verification

### Fill Profile Information

Enter the following:

1. **Organization name**: Organization name [Optional]
2. **Default region**: Choose database location
   - **us-east**: US East
   - **us-west**: US West
   - **eu-west**: EU West
   - **ap-southeast**: Asia Pacific

  src="https://res.cloudinary.com/demo/image/upload/v1/planetscale/planetscale-account-setup-step2-en.png"
  alt="Profile setup"
  caption="PlanetScale organization setup"
/>

<Callout type="info">
**Region Selection Tips**

- Choose region with most users for better speed
- Korea users: ap-southeast recommended
- Can be changed later
</Callout>

</Step>

<Step number={3} title="Create Your First Database">

1. In dashboard, click **Create a new database** or **New database** button
2. Enter database information:
   - **Database name**: Database name [e.g., myapp_db]
   - **Region**: Use default or change
   - **MySQL version**: 8.0 or 5.7 [8.0 recommended]

<CopyBlock
  code="Database naming rules:
- Start with lowercase
- Numbers and underscores allowed
- Examples: my_app_db, app_production, userdb_2024"
  language="bash"
/>

### Choose Branch

- **main**: Production branch [default]
- **develop**: Development branch [optional]

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/planetscale/planetscale-account-setup-step3-en.png"
  alt="Create database"
  caption="PlanetScale database creation form"
/>

3. Click **Create database** button.

<Callout type="tip">
üí° **PlanetScale Branching Concept**

- **main branch**: Production data
- **develop branch**: Development environment
- Like Git, test schema changes in multiple branches then merge

Example:
```
main branch       develop branch
[Production] <-- [Dev testing]
                  feature branch
                  [New feature dev]
```
</Callout>

</Step>

<Step number={4} title="Verify Database Connection Information">

After database creation, connection info appears.

### Copy Connection Info

1. Select created database in dashboard
2. Click **Connect** button
3. Following info displays:
   - **Host**: Database hostname
   - **Username**: Database username
   - **Password**: Password [newly generated]
   - **Database**: Database name

  src="https://res.cloudinary.com/demo/image/upload/v1/planetscale/planetscale-account-setup-step4-en.png"

### Test Connection

For production, regenerate password:

1. Go to **Settings** tab ‚Üí **Password**
2. Click **Regenerate password**

<DevTip>

- Each branch has independent password
- Regular regeneration recommended [every 3 months]
- Old password accessible for some time after regeneration
</DevTip>

## Troubleshooting

### "Invalid database name"

- Use only lowercase
- Only underscores allowed for special chars
- Already used names not allowed

### "Unable to create database"

- Check free account storage limit
- Delete unnecessary databases and retry

### Don't remember connection info?

- Click database in dashboard
- View again in **Connections** tab
