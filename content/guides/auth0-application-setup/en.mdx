<Step number={2} title="Configure Application Settings">

1. Click the **Settings** tab for your new application.
2. Note your **Client ID** and **Client Secret** (you'll need these for your code).
3. Set **Application URIs**:
   - **Allowed Callback URLs**: `http://localhost:3000/callback` (for development)
   - **Allowed Logout URLs**: `http://localhost:3000` (for development)
   - **Allowed Web Origins**: `http://localhost:3000` (for development)

<Step number={4} title="Initialize Auth0 in Your Application">

Here's a basic example using Node.js/Express. For other frameworks, see Auth0 docs:

<CopyBlock
  code="const { auth } = require('express-openid-connect');
const express = require('express');

const app = express();

// Configure Auth0
app.use(auth({
  authRequired: false,
  auth0Logout: true,
  secret: process.env.AUTH0_SECRET,
  baseURL: process.env.AUTH0_BASE_URL,
  clientID: process.env.AUTH0_CLIENT_ID,
  issuerBaseURL: process.env.AUTH0_ISSUER_BASE_URL,
}));

// Login route
app.get('/login', (req, res) = {
  res.oidc.login({ returnTo: '/' });
});

// Logout route
app.get('/logout', (req, res) = {
  res.oidc.logout({ returnTo: '/' });
});

// Protected route - requires login
app.get('/profile', (req, res) = {
  if (req.oidc.isAuthenticated()) {
    res.send('Welcome ' + req.oidc.user.name);
  } else {
    res.send('Please log in');
  }
});

app.listen(3000, () = console.log('App running on 3000'));"
  language="javascript"

## Frequently Asked Questions

<Callout type="info">
**Q. Where do user passwords get stored?**

A. Auth0 stores all user passwords securely encrypted in Auth0's database. Your app never sees the passwords - Auth0 handles all password hashing and verification.
</Callout>

<Callout type="info">
**Q. Can I customize the login page?**

A. Yes! You can customize Auth0's login page (Universal Login) with your branding, colors, and custom HTML. See the Customize section in Auth0 Dashboard.
</Callout>

<Callout type="info">
**Q. How do I get user information after login?**

A. Auth0 provides user information in `user` object after login. This includes name, email, picture, and custom user metadata you add.
</Callout>

<Callout type="info">
**Q. What if my app uses a different programming language?**

A. Auth0 supports virtually all popular languages (Python, PHP, Java, Ruby, Go, etc.). Find your official SDK in Auth0 documentation.
</Callout>

## Troubleshooting

### Login redirect doesn't work

- Verify your Callback URLs in Auth0 Settings match your code
- For development, use `http://localhost:3000/callback`
- For production, use `https://yourdomain.com/callback`
- Make sure you're using http (not https) for localhost

### Users not created

- Check that signup is enabled in Auth0 Dashboard
- Go to Authentication > Connections > Username-Password
- Ensure the Database connection is enabled for your application

### User data not showing

- Verify the user is logged in (check if `oidc.isAuthenticated()` is true)
- Try logging out and back in
- Check browser console for errors
