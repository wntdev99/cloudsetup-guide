# Docker 설치 및 설정 - 컨테이너 개발 시작하기

Docker는 애플리케이션과 그 의존성을 컨테이너라는 표준화된 단위로 패킹하는 도구입니다. 이 가이드를 따라 Docker Desktop을 설치하고 첫 번째 컨테이너를 실행해보세요!

## Docker란 무엇인가요?

<Callout type="info">
🐳 **Docker는 컨테이너화 플랫폼입니다**

Docker의 주요 기능:
- **환경 일관성**: 개발 중인 것과 동일한 환경에서 실행
- **빠른 배포**: 가상 머신보다 훨씬 빠른 시작 시간
- **리소스 효율**: 각 컨테이너는 경량이면서 독립적
- **버전 관리**: 이미지를 통해 정확한 버전 관리 가능
- **확장성**: 여러 컨테이너를 쉽게 관리

예를 들어:
- `nodejs-app`: Node.js 애플리케이션용
- `postgres-db`: PostgreSQL 데이터베이스용
- `nginx-server`: Nginx 웹 서버용
</Callout>

<Step number={1} title="Docker Desktop 다운로드">

### macOS 설치

1. [Docker 공식 웹사이트](https://www.docker.com/products/docker-desktop)에 접속합니다.
2. 상단의 **Download for Mac** 버튼을 클릭합니다.
3. Mac의 CPU에 따라 선택합니다:
   - **Apple Silicon [M1/M2/M3]**: 최신 Mac
   - **Intel Chip**: 구형 Mac

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/docker/docker-installation-step1-ko.png"
  alt="Docker Desktop 다운로드 페이지"
  caption="Docker 공식 사이트에서 Mac 버전 다운로드"
  priority={true}
/>

### Windows 설치

1. **Download for Windows** 버튼을 클릭합니다.
2. WSL 2 [Windows Subsystem for Linux]가 자동으로 설치됩니다.
3. Docker Desktop 설치 파일을 다운로드합니다.

<Callout type="warning">
⚠️ **Windows 요구사항**

- Windows 10 Pro, Enterprise, Education 이상
- 또는 Windows 11
- Hyper-V 지원이 필요합니다
</Callout>

### Linux 설치

1. Linux는 별도 설치 방식을 사용합니다.
2. 터미널을 열고 다음 명령어를 실행합니다:

<CopyBlock
  code="curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh"
  language="bash"
/>

</Step>

---

<Step number={2} title="Docker Desktop 설치 및 실행">

### macOS 및 Windows

1. 다운로드한 설치 파일을 실행합니다.
2. 설치 프로세스를 따릅니다 [약 2-3분 소요].
3. 설치가 완료되면 Applications 폴더에서 Docker를 찾아 실행합니다.
4. Docker 아이콘이 메뉴바에 나타나면 실행 중입니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/docker/docker-installation-step2-ko.png"
  alt="Docker 설치 진행 화면"
  caption="설치 진행 중 화면 - 관리자 권한을 요청합니다"
/>

<Callout type="tip">
💡 **팁**: Docker를 처음 실행하면 시스템 암호를 입력하라고 요청합니다. Docker는 Hypervisor [가상화 기술]를 사용하기 위해 관리자 권한이 필요합니다.
</Callout>

### Linux

설치 후 docker 명령어가 바로 사용 가능합니다. 사용자를 docker 그룹에 추가하려면:

<CopyBlock
  code="sudo usermod -aG docker $USER
newgrp docker"
  language="bash"
/>

</Step>

## Docker 설치 이후 확인 사항

### 시스템 요구사항 충족 확인

<CopyBlock
  code="docker ps
docker ps -a
docker images"
  language="bash"
/>

각 명령어의 의미:
- `docker ps`: 실행 중인 컨테이너 목록
- `docker ps -a`: 모든 컨테이너 목록 [중지된 것 포함]
- `docker images`: 설치된 이미지 목록

### Docker Hub 로그인 [선택사항]

나중에 이미지를 공유하려면 Docker Hub 계정을 만들고 로그인합니다:

<CopyBlock
  code="docker login"
  language="bash"
/>

</Callout>

## 문제 해결

### "Cannot connect to Docker daemon"

- Docker Desktop이 실행 중인지 확인
- Mac/Windows에서 메뉴바에서 Docker 아이콘 확인
- Docker를 다시 시작해보세요

### "Permission denied while trying to connect to Docker daemon" [Linux]

- 사용자를 docker 그룹에 추가: `sudo usermod -aG docker $USER`
- 터미널을 새로 시작하거나 `newgrp docker` 실행

### Windows에서 "WSL 2 installation is incomplete"

- Windows PowerShell을 관리자 권한으로 실행
- `wsl --install` 실행
- 시스템 재부팅
