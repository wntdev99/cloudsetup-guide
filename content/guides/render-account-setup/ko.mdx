# Render 계정 만들기 및 설정

Render는 **최신식 클라우드 배포 플랫폼**으로 GitHub에서 자동으로 배포할 수 있습니다. 90일 동안 완전히 무료로 사용하여 프로덕션 앱을 배포할 수 있습니다. 15분 안에 계정을 설정할 수 있습니다.

## 시작하기 전에

<Callout type="info">
🚀 **Render란?** Render는 GitHub에서 자동으로 배포되는 최신식 클라우드 플랫폼입니다. Node.js, Python, Go, Ruby 등 모든 언어를 지원합니다.
</Callout>

---

<Step number={1} title="Render 계정 생성">

1. [Render.com](https://render.com)을 방문합니다.
2. **가입하기** 버튼을 클릭합니다.
3. **GitHub로 로그인** (권장) 또는 이메일로 가입합니다.
4. GitHub을 선택하면 자동으로 로그인됩니다.
5. 이메일 가입을 선택하면 이메일과 비밀번호를 입력합니다.
6. **계정 생성**을 클릭합니다.

  src="https://res.cloudinary.com/demo/image/upload/v1/render/step1-signup-ko.png"

/>

<Callout type="tip">
💡 **팁**: GitHub로 가입하면 자동 배포가 더 간단합니다.
</Callout>

</Step>

---

<Step number={2} title="GitHub 리포지토리 연결">

1. Render 대시보드에서 **New +** 버튼을 클릭합니다.
2. **Web Service**를 선택합니다.
3. **GitHub 연결** 버튼을 클릭합니다 (처음 접속 시).
4. GitHub에서 Render 앱 권한을 승인합니다.
5. 배포할 **리포지토리**를 선택합니다.
6. **Connect**를 클릭합니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/render/step2-github-connect-ko.png"
  alt="GitHub 연결"
  caption="GitHub 리포지토리를 Render에 연결"
/>

<Callout type="info">
ℹ️ **참고**: 모든 GitHub 리포지토리에 접근을 허용하거나 선택한 리포지토리만 허용할 수 있습니다.
</Callout>

</Step>

---

<Step number={3} title="웹 서비스 배포 설정">

GitHub 리포지토리를 선택한 후:

1. **웹 서비스 이름** 입력 (예: my-awesome-app).
2. **런타임** 선택 (Node.js, Python, Go, Ruby 등).
3. **빌드 명령어** 입력 (예: npm install).
4. **시작 명령어** 입력 (예: npm start).
5. **인스턴스 타입** 선택:
   - Free (90일 무료)
   - 또는 Starter, Standard (유료)

6. **환경 변수** 추가 (필요시):
   - DATABASE_URL, API_KEY 등

7. **배포**를 클릭합니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/render/step3-web-service-ko.png"
  alt="웹 서비스 설정"
  caption="Render에서 웹 서비스 설정"
/>

<Callout type="tip">
💡 **팁**: render.yaml 파일을 리포지토리에 추가하여 설정을 자동화할 수 있습니다.
</Callout>

</Step>

---

<Step number={4} title="배포 및 상태 확인">

1. **배포** 버튼을 클릭하면 배포가 시작됩니다.
2. 로그를 실시간으로 볼 수 있습니다.
3. 배포가 완료되면 **URL**이 제공됩니다 (예: https://my-awesome-app.onrender.com).
4. 대시보드에서:
   - **배포 상태** 확인
   - **로그** 보기
   - **환경 변수** 관리
   - **수동 배포** 또는 **자동 배포** 설정

**자동 배포 설정:**

GitHub에 푸시하면 자동으로 배포됩니다.

1. **Auto-Deploy** 토글을 활성화합니다.
2. **메인 브랜치** (또는 다른 브랜치) 선택.
3. GitHub에 푸시하면 자동 배포가 됩니다.

## 개발자 팁

<DevTip>
**render.yaml 예제:**

```yaml
services:
  - type: web
    name: my-app
    runtime: node
    buildCommand: npm install
    startCommand: npm start
    envVars:
      - key: NODE_ENV
        value: production
```

리포지토리 루트에 저장하면 배포 설정이 자동화됩니다.
</DevTip>

## 자주 묻는 질문

<Callout type="info">
**Q. 90일 후에는 어떻게 되나요?**

A. 무료 인스턴스는 자동으로 중지됩니다. 계속 사용하려면 유료 플랜으로 업그레이드해야 합니다 (월 $7.99부터).
</Callout>

<Callout type="info">
**Q. 무료 인스턴스는 얼마나 강력한가요?**

A. 500MB RAM, 0.5 CPU로 소규모 프로젝트에 충분합니다. 프로덕션 트래픽이 많으면 업그레이드가 필요합니다.
</Callout>

<Callout type="info">
**Q. 데이터베이스는 어디에 두나요?**

A. Render의 PostgreSQL 서비스를 사용하거나, Supabase, Railway 등 외부 데이터베이스를 연결할 수 있습니다.
</Callout>

## 문제 해결

### 배포가 실패합니다

- 빌드 로그를 확인합니다
- buildCommand와 startCommand가 정확한지 확인합니다
- 의존성이 package.json/requirements.txt에 있는지 확인합니다

### 앱이 실행되지 않습니다

- 로그를 확인합니다 (대시보드 > Logs)
- 포트 설정을 확인합니다 (기본값 3000, 10000 또는 10001)
- 환경 변수가 올바른지 확인합니다
