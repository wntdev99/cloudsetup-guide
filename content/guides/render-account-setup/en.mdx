<Step number={1} title="Create Render Account">

1. Visit [Render.com](https://render.com).
2. Click **Sign up** button.
3. Choose **Sign up with GitHub** (recommended) or email.
4. GitHub will auto-login.
5. Email signup requires email and password.
6. Click **Create Account**.

<Step number={2} title="Connect GitHub Repository">

1. Click **New +** button in Render dashboard.
2. Select **Web Service**.
3. Click **Connect GitHub** (first time).
4. Authorize Render app in GitHub.
5. Select your **repository** to deploy.
6. Click **Connect**.

  src="https://res.cloudinary.com/demo/image/upload/v1/render/step2-github-connect-en.png"

<Step number={3} title="Configure Web Service Deployment">

After selecting your GitHub repository:

1. Enter **web service name** (e.g., my-awesome-app).
2. Select **runtime** (Node.js, Python, Go, Ruby, etc.).
3. Enter **build command** (e.g., npm install).
4. Enter **start command** (e.g., npm start).
5. Select **instance type**:
   - Free (90-day free)
   - Or Starter, Standard (paid)

6. Add **environment variables** (if needed):
   - DATABASE_URL, API_KEY, etc.

7. Click **Deploy**.

  src="https://res.cloudinary.com/demo/image/upload/v1/render/step3-web-service-en.png"

</Step>

---

<Step number={4} title="Deploy and Check Status">

1. Click **Deploy** to start deployment.
2. Watch logs in real time.
3. When complete, you get a **URL** (e.g., https://my-awesome-app.onrender.com).
4. In dashboard:
   - Check **deployment status**
   - View **logs**
   - Manage **environment variables**
   - Set **manual** or **auto-deploy**

**Enable auto-deploy:**

GitHub pushes auto-deploy your app.

1. Enable **Auto-Deploy** toggle.
2. Select **main branch** (or other).
3. GitHub push triggers auto-deploy.

## Developer Tips

<DevTip>
**render.yaml Example:**

```yaml
services:
  - type: web
    name: my-app
    runtime: node
    buildCommand: npm install
    startCommand: npm start
    envVars:
      - key: NODE_ENV
        value: production
```

Save in repo root to automate configuration.
</DevTip>

## Frequently Asked Questions

<Callout type="info">
**Q. What happens after 90 days?**

A. Free instance auto-stops. Upgrade to paid plan ($7.99/month) to continue.
</Callout>

<Callout type="info">
**Q. How powerful is the free instance?**

A. 500MB RAM, 0.5 CPU - enough for small projects. Upgrade for heavy production traffic.
</Callout>

<Callout type="info">
**Q. Where do I put my database?**

A. Use Render's PostgreSQL or connect external like Supabase or Railway.

## Troubleshooting

### Deployment fails

- Check build logs
- Verify buildCommand and startCommand
- Check dependencies in package.json/requirements.txt

### App won't run

- Check logs (Dashboard > Logs)
- Verify port (default 3000, 10000, or 10001)
- Check environment variables
