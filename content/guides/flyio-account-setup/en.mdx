<Step number={2} title="Install CLI (macOS/Linux)">

You need to install the flyctl CLI for Fly.io deployment.

**macOS (Homebrew recommended):**

<CopyBlock
  code="brew install flyctl"
  language="bash"
/>

**Or using curl:**

<CopyBlock
  code="curl -L https://fly.io/install.sh | sh"
  language="bash"
/>

**Linux (Ubuntu/Debian):**

<CopyBlock
  code="curl -L https://fly.io/install.sh | sh"
  language="bash"
/>

Add to PATH:

<CopyBlock
  code="export PATH=$PATH:~/.fly/bin"
  language="bash"
/>

Add to .bashrc or .zshrc for permanent setup:

<CopyBlock
  code="echo 'export PATH=$PATH:~/.fly/bin' >> ~/.bashrc"
  language="bash"
/>

**Verify installation:**

<CopyBlock
  code="flyctl version"
  language="bash"
/>

<Step number={4} title="Connect CLI to Fly.io Account">

Now connect flyctl to your Fly.io account.

<CopyBlock
  code="flyctl auth login"
  language="bash"
/>

When you run the command:

1. An **authentication URL** is displayed.
2. Open the URL in your browser or press **"Press Enter to open browser"**.
3. Log in with your Fly.io account.
4. Click **Authorize**.
5. Return to the terminal and authentication is complete.

**Verify authentication:**

<CopyBlock
  code="flyctl auth whoami"
  language="bash"
/>

Success if your email address is printed.

  src="https://res.cloudinary.com/demo/image/upload/v1/flyio/step4-auth-login-en.png"

## Frequently Asked Questions

<Callout type="info">
**Q. What can I do with Fly.io's free tier?**

A. You get 3 shared CPU VMs and 3GB monthly storage for free. Enough for small projects and development.
</Callout>

<Callout type="info">
**Q. Do I need Docker?**

A. Fly.io uses Docker, but flyctl automatically generates Dockerfile for you, so you don't need to write one.
</Callout>

<Callout type="info">
**Q. Can I connect a domain?**

A. Yes! Register a domain with Fly.io or connect an existing domain via CNAME or DNS settings.
</Callout>

<Callout type="info">
**Q. Where should I put my database?**

A. Use Fly.io's Postgres service or connect external databases like Supabase or Railway.
</Callout>

## Troubleshooting

### "command not found: flyctl" error

- Verify flyctl is added to PATH
- Restart your terminal
- Set PATH manually

### Authentication fails

- Log out and try again: `flyctl auth logout && flyctl auth login`
- Check internet connection
- Check Fly.io status page

### flyctl version doesn't work

- Verify installation is complete
- Try restarting your system
- Reinstall flyctl
