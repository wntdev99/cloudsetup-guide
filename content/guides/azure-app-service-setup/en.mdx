# Azure App Service Setup - Deploy Web Apps

Azure App Service is a **fully managed web hosting service** supporting Node.js, Python, .NET, Java, PHP, and more. It offers multiple deployment methods (GitHub, Local Git, Docker) and guarantees automatic scaling and high availability. This guide shows how to create and deploy a web app.

<Step number={1} title="Create App Service">

Create a web app in Azure Portal.

1. Sign in to [Azure Portal](https://portal.azure.com)
2. Click **"Create a resource"**
3. Search for **"App Service"**
4. Select **"App Service"** (Microsoft publisher)
5. Click **"Create"**

<Step number={2} title="Configure Basic Settings">

Set up web app basic information.

1. Select your **Subscription** (keep default)
2. Create or select **Resource Group**
3. Enter **Name**:
   - Example: "mywebapp-2026"
   - Must be globally unique (".azurewebsites.net" auto-added)
4. Select **Publish** method:
   - **"Code"**: Deploy source code
   - **"Docker Container"**: Deploy container image
5. Choose **Runtime Stack**:
   - **Node.js**: JavaScript web app
   - **Python**: Python application
   - **.NET**: C# application
   - **PHP**: PHP application
   - **Java**: Java application
6. Select **Version** (latest recommended)

<DevTip>
Your app name becomes the URL, so choose a memorable, project-representative name.
</DevTip>

</Step>

<Step number={3} title="Configure App Service Plan">

Select the App Service plan allocating computing resources.

1. Select **App Service Plan**:
   - Click **"Create new"** (create new plan)
   - Or select existing plan
2. Enter **Plan name** (e.g., "myappserviceplan")
3. Select **Sku and size**:
   - **Free tier (F1)**: 60 minutes/month (testing only)
   - **Shared (D1)**: 24 hours (small apps)
   - **Basic (B1)**: Basic performance (production recommended)
4. For free tier, select "Change size" > "F1 Free"

<Step number={4} title="Configure Deployment">

Set up deployment source and continuous deployment.

1. Go to **"Deployment"** tab
2. Select **Deployment source**:
   - **GitHub**: Auto-deploy from GitHub repo
   - **Azure DevOps**: Azure DevOps pipelines
   - **Local Git**: Deploy via local Git push
   - **FTP**: Upload files via FTP
3. If **GitHub** selected:
   - Authenticate GitHub account
   - Select repository to deploy
   - Choose branch (e.g., "main")
4. Enable **Continuous deployment** (if desired)

<Step number={5} title="Deploy Web App and Access">

Deploy the web app and access it.

1. Click **"Create"** to complete App Service creation
2. Click **"Go to resource"**
3. In **App Service dashboard**:
   - Click **"Browse"** in Essentials to view web app
   - Or copy **URL** and access in browser
4. **Check deployment status**:
   - Go to **"Deployments"** menu to view history
   - Go to **"Logs"** menu to view deployment logs

## Troubleshooting

<DevTip>
**"502 Bad Gateway or error page"**

Check the following:
1. View application logs (**"Log stream"** menu)
2. Verify runtime version matches application
3. Check required environment variables are set
4. Confirm port is 80 or 8080
5. Verify dependencies are installed properly
</DevTip>

<Callout type="warning">
‚ö†Ô∏è **Free Tier Usage Notes**

- **60-minute limit**: Only 60 minutes/month operation (auto-stops after)
- **Periodic restart**: Auto-stops after 20 minutes inactivity
- **Performance**: Shared resources cause slower performance
- **Downtime**: Scheduled maintenance causes outages
- **Upgrade**: B1+ tier recommended for better experience
</Callout>
