# Azure App Service 설정 - 웹앱 배포

Azure App Service는 **완전히 관리되는 웹 호스팅 서비스**로, Node.js, Python, .NET, Java, PHP 등 다양한 언어와 프레임워크를 지원합니다. GitHub, Local Git, Docker 등 여러 배포 방식을 제공하며, 자동 스케일링과 높은 가용성을 보장합니다. 이 가이드에서는 웹앱을 생성하고 배포합니다.

## 시작하기 전에

<Callout type="info">
🌐 **Azure App Service의 특징**

- **다중 언어**: Node.js, Python, .NET, Java, PHP 등
- **자동 배포**: GitHub, Azure DevOps와 연동
- **자동 스케일링**: 트래픽에 따라 자동 확장
- **보안**: SSL/TLS, 방화벽, 인증 지원
- **모니터링**: Application Insights 통합
</Callout>

<Step number={1} title="App Service 생성">

Azure Portal에서 웹앱을 생성합니다.

1. [Azure Portal](https://portal.azure.com)에 로그인
2. **"리소스 만들기"** 클릭
3. **"App Service"** 검색
4. **"App Service"** 선택 (Microsoft 게시자)
5. **"만들기"** 클릭

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/azure/azure-app-service-setup-step1-ko.png"
  alt="App Service 생성 시작"
  caption="Azure Portal에서 App Service 생성을 시작합니다"
/>

<Callout type="tip">
💡 **팁**: "빠른 만들기" 옵션으로 빠르게 웹앱을 생성할 수 있습니다.
</Callout>

</Step>

---

<Step number={2} title="기본 설정 구성">

웹앱의 기본 정보를 설정합니다.

1. **구독** 선택 (기본값 유지)
2. **리소스 그룹** 생성 또는 선택
3. **이름** 입력:
   - 예: "mywebapp-2026"
   - 전역적으로 고유해야 함 (자동으로 ".azurewebsites.net" 추가)
4. **게시** 방식 선택:
   - **"코드"**: 소스 코드 배포
   - **"Docker 컨테이너"**: 컨테이너 이미지 배포
5. **런타임 스택** 선택:
   - **Node.js**: JavaScript 웹앱
   - **Python**: Python 애플리케이션
   - **.NET**: C# 애플리케이션
   - **PHP**: PHP 애플리케이션
   - **Java**: Java 애플리케이션
6. **버전** 선택 (최신 권장)

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/azure/azure-app-service-setup-step2-ko.png"
  alt="기본 설정 화면"
  caption="웹앱의 기본 정보와 런타임 스택을 설정합니다"
/>

<DevTip>
웹앱 이름은 URL이 되므로, 기억하기 쉽고 프로젝트를 나타내는 이름을 선택하세요.
</DevTip>

</Step>

---

<Step number={3} title="App Service 계획 설정">

컴퓨팅 리소스를 할당하는 App Service 계획을 선택합니다.

1. **App Service 계획** 선택:
   - **"새로 만들기"** 클릭 (새 계획 생성)
   - 또는 기존 계획 선택
2. **계획 이름** 입력 (예: "myappserviceplan")
3. **Sku 및 크기** 선택:
   - **무료 계층 (F1)**: 월간 60분 (테스트 전용)
   - **공유 (D1)**: 24시간 (소규모 앱)
   - **기본 (B1)**: 기본 성능 (프로덕션 권장)
4. 무료 계층 선택 시 "크기 변경" > "F1 무료" 선택

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/azure/azure-app-service-setup-step3-ko.png"
  alt="App Service 계획 설정 화면"
  caption="App Service 계획을 선택하고 가격 책정 계층을 구성합니다"
/>

<Callout type="warning">
⚠️ **무료 계층의 제한사항**

- **월간 60분 컴퓨팅 시간**: 제한된 운영 시간
- **공유 리소스**: 다른 앱과 서버 공유
- **기능 제한**: 커스텀 도메인, SSL, 자동 스케일 미지원
- **권장**: 개발/테스트용 (프로덕션은 B1 이상)
</Callout>

</Step>

---

<Step number={4} title="배포 설정">

배포 소스와 자동 배포를 설정합니다.

1. **"배포"** 탭으로 이동
2. **배포 소스** 선택:
   - **GitHub**: GitHub 리포지토리에서 자동 배포
   - **Azure DevOps**: Azure DevOps 파이프라인
   - **로컬 Git**: 로컬에서 Push로 배포
   - **FTP**: FTP 접속으로 파일 업로드
3. **GitHub 선택 시**:
   - GitHub 계정 인증
   - 배포할 리포지토리 선택
   - 브랜치 선택 (예: "main")
4. **자동 배포** 활성화 (선택 시)

<Step number={5} title="웹앱 배포 및 접근">

웹앱을 배포하고 접근합니다.

1. **"만들기"** 클릭하여 App Service 생성 완료
2. **리소스로 이동** 클릭
3. **App Service 대시보드**에서:
   - **기본 사항**의 **"찾아보기"** 클릭하여 웹앱 확인
   - 또는 **URL** 복사하여 브라우저에 접속
4. **배포 상태 확인**:
   - **"배포"** 메뉴에서 배포 히스토리 확인
   - **"로그"** 메뉴에서 배포 로그 확인

## 문제 해결

<DevTip>
**"502 Bad Gateway 또는 에러 페이지"**

다음을 확인하세요:
1. 애플리케이션 로그 확인 (**"로그 스트림"** 메뉴)
2. 런타임 버전이 애플리케이션과 호환되는지 확인
3. 필수 환경 변수 설정 확인
4. 포트가 80 또는 8080인지 확인
5. 의존성이 제대로 설치되었는지 확인
</DevTip>

<Callout type="warning">
⚠️ **무료 계층 사용 시 주의사항**

- **60분 제한**: 월간 60분만 실행 (초과하면 자동 중지)
- **정기적 재시작**: 20분 비활동 후 자동 중지
- **성능**: 공유 리소스로 인해 느림
- **다운타임**: 정기 유지보수로 인한 가동 중지
- **업그레이드**: B1 이상으로 업그레이드 권장
</Callout>
