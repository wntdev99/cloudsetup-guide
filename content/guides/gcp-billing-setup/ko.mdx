# GCP 예산 알림 설정 - 크레딧 초과 방지하기

$300의 무료 크레딧을 받았다면, 이제 **예산 알림**을 설정해서 크레딧이 예상치 못하게 소진되는 것을 방지해야 합니다. 이 가이드를 따라 10분 안에 안전장치를 만들어보세요.

## 왜 예산 알림이 필요한가요?

<Callout type="warning">
⚠️ **실수로 큰 비용이 발생할 수 있습니다**

GCP는 강력한 클라우드 서비스이지만, 실수로 리소스를 켜두거나 잘못된 설정을 하면 크레딧이 빠르게 소진될 수 있습니다. 예산 알림을 설정하면:
- 크레딧 사용량이 50%, 80%, 100%에 도달하면 자동으로 이메일 알림
- 예상치 못한 비용 발생 시 조기 감지
- 크레딧 소진 전에 조치를 취할 수 있는 시간 확보
</Callout>

<Step number={1} title="결제 페이지 접속">

1. [Google Cloud Console](https://console.cloud.google.com)에 로그인합니다.
2. 왼쪽 메뉴에서 **≡** (햄버거 메뉴)를 클릭합니다.
3. **결제(Billing)** 메뉴를 클릭합니다.

  src="https://res.cloudinary.com/demo/image/upload/gcp-billing-setup-step1-ko.png"

<Callout type="tip">
💡 **팁**: 결제 메뉴가 보이지 않나요? 상단 검색창에 "결제"를 입력하면 빠르게 찾을 수 있습니다.
</Callout>

</Step>

---

<Step number={2} title="예산 및 알림 페이지로 이동">

1. 결제 페이지에서 왼쪽 사이드바의 **예산 및 알림(Budgets & alerts)**을 클릭합니다.
2. 상단의 **예산 만들기(CREATE BUDGET)** 버튼을 클릭합니다.

  src="https://res.cloudinary.com/demo/image/upload/gcp-billing-setup-step2-ko.png"

/>

<Step number={3} title="예산 설정">

### 3-1. 범위 선택

1. **결제 계정**: 현재 계정이 자동으로 선택됩니다.
2. **프로젝트**:
   - "모든 프로젝트" 선택 (권장)
   - 또는 특정 프로젝트만 선택 가능
3. **서비스**: "모든 서비스" 선택 (권장)
4. **계속** 버튼 클릭

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/gcp-billing-setup-step3a-ko.png"
  alt="예산 범위 설정"
  caption="모든 프로젝트와 서비스를 포함하도록 설정하세요"
/>

### 3-2. 예산 금액 설정

1. **예산 유형**: "지정된 금액" 선택
2. **대상 금액**: **$300** 입력
   - 무료 크레딧 전액을 예산으로 설정
3. **계속** 버튼 클릭

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/gcp-billing-setup-step3b-ko.png"
  alt="예산 금액 설정"
  caption="$300로 설정하여 무료 크레딧 전체를 모니터링하세요"
/>

<DevTip>
**왜 $300인가요?**

무료 크레딧이 $300이므로, 예산도 $300로 설정합니다. 이렇게 하면 크레딧 사용률(%)을 정확히 추적할 수 있습니다.
</DevTip>

</Step>

---

<Step number={4} title="알림 임계값 설정">

### 4-1. 알림 규칙 설정

기본적으로 다음 임계값에서 알림이 전송됩니다:
- ✅ **50%** ($150 사용 시)
- ✅ **90%** ($270 사용 시)
- ✅ **100%** ($300 사용 시)

추가 알림을 설정하려면:
1. **알림 임계값 규칙 추가** 클릭
2. 원하는 퍼센트 입력 (예: 80%)
3. 필요한 만큼 반복

1. **이메일 알림 대상자** 섹션에서:
   - "결제 계정 관리자 및 사용자"가 기본으로 체크됨
   - 추가 이메일 주소를 입력할 수도 있습니다

2. **마침** 버튼을 클릭합니다.

## 추가 보안 설정 (선택사항)

### Cloud Pub/Sub 알림 (고급)

더 빠른 알림을 원한다면 Pub/Sub을 설정할 수 있습니다:
1. 예산 수정 페이지에서 "Pub/Sub 주제에 연결" 선택
2. 주제를 생성하고 연결
3. Cloud Functions나 다른 서비스와 통합 가능

<DevTip>
이 방법은 고급 사용자를 위한 것입니다. 이메일 알림만으로도 대부분의 경우 충분합니다.
</DevTip>

## 자주 묻는 질문

<Callout type="info">
**Q. 알림을 받으면 자동으로 서비스가 중지되나요?**

A. 아니요. 알림은 **경고만** 제공합니다. 서비스는 계속 작동하며, 수동으로 중지하거나 리소스를 삭제해야 합니다. 무료 체험 계정은 크레딧이 소진되면 자동으로 중지됩니다.
</Callout>

<Callout type="info">
**Q. 여러 개의 예산을 만들 수 있나요?**

A. 네! 프로젝트별, 서비스별로 여러 예산을 만들 수 있습니다. 예를 들어:
- 전체 계정: $300
- Vision API만: $50
- Compute Engine만: $100
</Callout>

<Callout type="info">
**Q. 알림 임계값을 나중에 수정할 수 있나요?**

A. 네! 예산 목록에서 예산을 클릭하고 수정 버튼을 누르면 언제든지 변경할 수 있습니다.
</Callout>

## 문제 해결

### 알림 이메일이 오지 않아요

1. **스팸 폴더 확인**: Gmail의 스팸 폴더를 확인하세요
2. **이메일 주소 확인**: 결제 계정에 등록된 이메일이 맞는지 확인
3. **시간 대기**: 첫 알림은 설정 후 최대 24시간이 걸릴 수 있습니다
4. **테스트**: 아주 작은 리소스를 생성해서 비용이 발생하는지 확인

### 예산을 만들 수 없어요

- 결제 계정 권한이 있는지 확인하세요
- 브라우저 캐시를 지우고 다시 시도하세요
- 다른 브라우저나 시크릿 모드로 시도하세요

### 크레딧 사용량이 업데이트되지 않아요

- 크레딧 사용량은 **최대 24시간 지연**될 수 있습니다
- 실시간이 아니므로 여유를 두고 예산을 설정하세요

## 비용 관리 팁

<Callout type="tip">
💰 **크레딧을 아끼는 방법**

1. **사용하지 않는 리소스 삭제**: VM, 디스크, 로드밸런서 등
2. **개발 환경은 작게**: 최소 사양으로 시작
3. **자동 종료 설정**: Compute Engine 인스턴스에 일정 설정
4. **무료 티어 활용**: 많은 서비스가 월별 무료 한도 제공
5. **정기적으로 확인**: 매주 결제 대시보드 확인
</Callout>
