# SendGrid 계정 만들기 및 발신자 인증 설정

SendGrid는 **강력한 클라우드 기반 이메일 전송 서비스**로 개발자가 대량의 이메일을 쉽게 보낼 수 있도록 도와줍니다. 매월 100개 이메일을 완전히 무료로 보낼 수 있습니다. 15분 안에 계정을 설정하고 첫 이메일을 보낼 준비를 할 수 있습니다.

<Callout type="info">
📧 **SendGrid란?** SendGrid는 SMTP 서버나 이메일 API를 통해 대량의 이메일을 안정적으로 전송하는 서비스입니다. 웹 애플리케이션에서 회원 확인, 비밀번호 재설정, 뉴스레터 등을 보낼 때 사용합니다.
</Callout>

---

<Step number={1} title="SendGrid 계정 생성">

1. [SendGrid.com](https://sendgrid.com)을 방문합니다.
2. **가입하기** 버튼을 클릭합니다.
3. **이메일 주소**를 입력합니다.
4. **암호**를 설정합니다 (강력한 암호 권장).
5. **이름**과 **성**을 입력합니다.
6. **회사 (선택사항)**를 입력할 수 있습니다.
7. **약관에 동의**하고 **계정 생성**을 클릭합니다.

💡 **팁**: 가입 후 이메일 확인이 필요합니다. 이메일을 확인하세요.
</Callout>

</Step>

<Step number={2} title="이메일 주소 확인">

1. 가입 후 SendGrid에서 **확인 이메일**을 보냅니다.
2. 이메일 받은편지함에서 SendGrid 이메일을 찾습니다.
3. **이메일 확인** 링크를 클릭합니다.
4. 확인이 완료되면 대시보드로 리다이렉트됩니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/sendgrid/step2-verify-email-ko.png"
  alt="SendGrid 이메일 확인"
  caption="확인 링크를 클릭하여 이메일을 확인합니다"
/>

<Callout type="warning">
⚠️ **주의**: 이메일 확인이 완료되지 않으면 API를 사용할 수 없습니다.
</Callout>

</Step>

---

<Step number={3} title="API 키 생성">

SendGrid에서 이메일을 보내려면 API 키가 필요합니다.

1. 대시보드에서 왼쪽 메뉴의 **설정** 섹션으로 이동합니다.
2. **API 키** 또는 **API Keys**를 클릭합니다.
3. **Create API Key** 버튼을 클릭합니다.
4. **API 키 이름**을 입력합니다 (예: "My App API Key").
5. **권한 수준**을 선택합니다:
   - **제한됨**: 특정 권한만 부여 (권장)
   - **전체 액세스**: 모든 권한 부여

6. **필수 권한**:
   - Mail Send
   - Contacts Import
   - Bounce (선택사항)

7. **Create & View** 버튼을 클릭합니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/sendgrid/step3-api-key-ko.png"
  alt="SendGrid API 키 생성"
  caption="새 API 키를 생성하세요 - 안전하게 저장하세요"
/>

**API 키를 복사하여 안전한 곳에 저장하세요:**

생성된 API 키는 **한 번만 표시됩니다**. 분실하면 새로 생성해야 합니다.

<Callout type="warning">
⚠️ **보안**: API 키를 절대 공개하거나 버전 관리에 커밋하지 마세요. 환경 변수에 저장하세요.
</Callout>

</Step>

---

<Step number={4} title="발신자 인증 설정">

SendGrid에서 이메일을 보내려면 발신자를 인증해야 합니다.

**간단한 인증 (권장 - 빠름):**

1. 대시보드에서 **설정** 메뉴로 이동합니다.
2. **발신자 인증** 또는 **Sender Authentication**을 클릭합니다.
3. **싱글 센더 인증** (Single Sender Verification) 선택.
4. **발신자 이메일 주소**를 입력합니다 (예: noreply@example.com).
5. **발신자 이름**을 입력합니다 (예: My App).
6. **회신 이메일** (선택사항)을 입력합니다.
7. **Create**를 클릭합니다.
8. SendGrid에서 **확인 이메일**을 보냅니다.
9. 이메일에서 **확인 링크**를 클릭합니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/sendgrid/step4-sender-verification-ko.png"
  alt="SendGrid 발신자 인증"
  caption="발신자 이메일 주소를 인증합니다"
/>

**도메인 인증 (프로덕션용 - 권장):**

도메인을 인증하면 더 높은 전송률을 얻을 수 있습니다.

1. **발신자 인증** 페이지에서 **도메인 인증** (Domain Authentication) 선택.
2. **도메인**을 입력합니다 (예: example.com).
3. **기본 인증** (Default to Domain) 선택 (권장).
4. **DNS 레코드**가 표시됩니다:
   - **CNAME** 레코드 (3개)
   - 또는 **MX** 레코드

5. **DNS 호스팅 제공자**에 로그인합니다:
   - GoDaddy, Namecheap, Route 53 등
6. **DNS 설정**에 레코드를 추가합니다.
7. SendGrid에서 **확인** 버튼을 클릭합니다.
8. DNS 레코드가 확인되면 완료됩니다.

## 개발자 팁

<DevTip>
**API 키 환경 변수 설정:**

.env 파일에 저장하세요:

```
SENDGRID_API_KEY=SG.xxxxxxxxxxxxxxxxxxxx
SENDGRID_FROM_EMAIL=noreply@example.com
SENDGRID_FROM_NAME=My App
```

Node.js에서 접근:

```javascript
const apiKey = process.env.SENDGRID_API_KEY;
const fromEmail = process.env.SENDGRID_FROM_EMAIL;
```
</DevTip>

## 자주 묻는 질문

<Callout type="info">
**Q. 월 100개 이메일이 도착하지 않으면 어떻게 되나요?**

A. 월 100개를 초과하면 추가 이메일은 전송되지 않습니다. 더 많이 보내려면 SendGrid 유료 플랜으로 업그레이드해야 합니다.
</Callout>

<Callout type="info">
**Q. API 키를 잃어버렸습니다. 어떻게 하나요?**

A. 새 API 키를 생성하면 됩니다. 대시보드의 API 키 페이지에서 "Create API Key" 버튼을 클릭하세요.
</Callout>

<Callout type="info">
**Q. 발신자 인증과 도메인 인증의 차이는?**

A. 발신자 인증은 특정 이메일 주소를 확인하며 빠릅니다. 도메인 인증은 전체 도메인을 확인하여 더 높은 전송률을 제공합니다.
</Callout>

<Callout type="info">
**Q. 스팸으로 표시되지 않으려면 어떻게 해야 하나요?**

A. SPF, DKIM, DMARC 레코드를 DNS에 추가하세요. SendGrid는 도메인 인증 시 이 레코드들을 자동으로 제공합니다.
</Callout>

## 문제 해결

### 이메일 확인이 도착하지 않습니다

- 스팸 폴더를 확인합니다
- 올바른 이메일 주소를 사용했는지 확인합니다
- 다시 확인 이메일을 요청합니다
- SendGrid 지원팀에 문의합니다

### API 키가 작동하지 않습니다

- API 키가 정확하게 복사되었는지 확인합니다
- API 키에 "Mail Send" 권한이 있는지 확인합니다
- 새 API 키를 생성해봅니다

### 발신자 인증이 거부되었습니다

- 유효한 이메일 주소를 사용했는지 확인합니다
- 도메인 인증을 시도해봅니다
- SendGrid 지원팀에 문의합니다
