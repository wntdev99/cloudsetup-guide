# Firebase Authentication 설정 - 사용자 인증 구현

Firebase Authentication은 **사용자 가입 및 로그인을 관리**하는 서비스입니다. 이메일/비밀번호, Google, Facebook 등 다양한 인증 방식을 지원하며, 사용자 관리와 보안을 자동으로 처리합니다. 이 가이드에서는 Firebase Authentication을 설정하고 다양한 인증 방식을 활성화합니다.

<Callout type="info">
🔐 **Firebase Authentication의 보안**

Firebase Authentication은 다음을 자동으로 처리합니다:
- 비밀번호 암호화 및 안전한 저장
- 세션 관리 및 토큰 생성
- 부정 행위 탐지
- 다중 인증(MFA) 지원
- 접근 로그 추적

<Step number={1} title="Authentication 서비스 활성화">

Firebase 콘솔에서 Authentication을 활성화합니다.

1. Firebase 콘솔 좌측 메뉴에서 **"Authentication"** 선택
2. **"Authentication 시작"** 또는 **"로그인 방법 설정"** 클릭
3. 사용 가능한 로그인 제공자 목록이 표시됩니다:
   - 이메일/비밀번호
   - Google
   - Facebook
   - GitHub
   - Apple
   - Microsoft
   - 전화번호
   - 익명

/>

<Callout type="tip">

</Callout>

</Step>

---

<Step number={2} title="이메일/비밀번호 인증 설정">

가장 기본적인 인증 방식을 활성화합니다.

1. 제공자 목록에서 **"이메일/비밀번호"** 선택
2. **"사용 설정"** 토글 클릭
3. **"이메일 링크(비밀번호 없음)"** 옵션:
   - 활성화하면 사용자가 이메일 링크로 로그인 가능
   - 비활성화하면 비밀번호 필수
4. **"저장"** 클릭

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/firebase/firebase-auth-setup-step2-ko.png"
  alt="이메일/비밀번호 인증 설정"
  caption="이메일/비밀번호 인증을 활성화하고 옵션을 설정합니다"
/>

<DevTip>
초기 단계에서는 "이메일 링크" 옵션을 비활성화하고 일반 비밀번호 방식을 사용하는 것이 좋습니다. 사용자가 더 익숙합니다.
</DevTip>

</Step>

<Step number={3} title="Google 로그인 추가 (선택사항)">

사용자가 Google 계정으로 로그인할 수 있게 합니다.

1. 제공자 목록에서 **"Google"** 선택
2. **"사용 설정"** 토글 클릭
3. **"프로젝트 지원 이메일"** 선택 (필수)
4. **"저장"** 클릭

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/firebase/firebase-auth-setup-step3-ko.png"
  alt="Google 로그인 설정"
  caption="Google 로그인을 활성화하고 지원 이메일을 설정합니다"
/>

<Callout type="info">
ℹ️ **Google OAuth 설정**

Firebase는 자동으로 Google OAuth를 설정합니다. 추가 구성은 필요하지 않습니다. Google 계정이 있는 사용자는 한 번의 클릭으로 로그인할 수 있습니다.
</Callout>

</Step>

---

<Step number={4} title="Facebook 로그인 추가 (선택사항)">

Facebook 계정으로도 로그인할 수 있게 합니다.

1. 제공자 목록에서 **"Facebook"** 선택
2. **"사용 설정"** 토글 클릭
3. **"앱 ID"** 및 **"앱 시크릿"** 입력 (Facebook 개발자 계정 필요)
   - Facebook Developers(https://developers.facebook.com)에서 앱 생성
   - 앱 설정에서 ID와 시크릿 복사
4. **"승인된 리다이렉트 URI"** 확인 (Firebase가 자동 생성)
5. **"저장"** 클릭

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/firebase/firebase-auth-setup-step4-ko.png"
  alt="Facebook 로그인 설정"
  caption="Facebook 앱 ID와 시크릿을 입력하여 Facebook 로그인을 활성화합니다"
/>

<Callout type="warning">
⚠️ **Facebook 개발자 계정 필수**

Facebook 로그인을 사용하려면:
1. Facebook Developers 계정 생성
2. 새 앱 생성 및 App ID, App Secret 발급
3. 앱 설정에 Firebase 리다이렉트 URI 등록
4. 앱이 "Live" 상태여야 함
</Callout>

</Step>

---

<Step number={5} title="사용자 관리 및 보안 설정">

Authentication 설정을 완료하고 추가 보안 옵션을 구성합니다.

1. **"Users"** 탭에서 가입한 사용자 목록 확인
2. **"Settings"** 탭에서 암호 정책 설정:
   - 최소 암호 길이 설정
   - 서드파티 쿠키 정책
3. **"User actions"** 설정:
   - 이메일 검증 메시지 커스터마이징
   - 비밀번호 재설정 메시지
   - 새 사용자 가입 메시지

import { initializeApp } from 'firebase/app';

## 문제 해결

<DevTip>
**"로그인 시 '인증 실패' 오류"**

이 오류가 나타나면:
1. Firebase 콘솔에서 Authentication이 활성화되었는지 확인
2. 사용자 계정이 Users 탭에 표시되는지 확인
3. 이메일과 비밀번호가 정확한지 재확인
4. CORS 설정이 올바른지 확인
5. Firebase 보안 규칙을 검토 (특히 Firestore과 Storage)
</DevTip>

<Callout type="warning">
⚠️ **보안 권장사항**

1. **클라이언트 측**: API 키를 코드에 하드코딩하지 마세요
2. **비밀번호**: 사용자에게 강력한 비밀번호를 권장하세요
3. **HTTPS**: 항상 HTTPS를 사용하세요
4. **세션**: 세션 토큰을 안전하게 관리하세요
5. **로그 아웃**: 민감한 작업 후 로그아웃 강제하기
</Callout>

## Firebase Authentication 제공자 비교

| 제공자 | 설정 난이도 | 사용자 편의성 | 추천 상황 |
|--------|-----------|------------|----------|
| 이메일/비밀번호 | 낮음 | 중간 | 모든 앱 (기본) |
| Google | 낮음 | 높음 | Google 사용자 대상 |
| Facebook | 중간 | 높음 | Facebook 사용자 대상 |
| Apple | 중간 | 높음 | iOS 앱 |
| 익명 | 낮음 | 낮음 | 임시 사용자 |
