# Firebase Authentication Setup - Implement User Authentication

Firebase Authentication is a **service that manages user sign-up and login**. It supports various authentication methods including email/password, Google, Facebook, and more, while automatically handling user management and security. This guide shows you how to set up Firebase Authentication and enable multiple sign-in methods.

<Step number={1} title="Enable Authentication Service">

Activate Authentication in the Firebase Console.

1. In the Firebase Console left menu, select **"Authentication"**
2. Click **"Get Started"** or **"Sign-in method"**
3. A list of available sign-in providers appears:
   - Email/Password
   - Google
   - Facebook
   - GitHub
   - Apple
   - Microsoft
   - Phone Number
   - Anonymous

<Step number={2} title="Set Up Email/Password Authentication">

Enable the most basic authentication method.

1. Select **"Email/Password"** from the provider list
2. Click the **"Enable"** toggle
3. **"Email link sign-in (passwordless)"** option:
   - When enabled, users can sign in via email link
   - When disabled, password is required
4. Click **"Save"**

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/firebase/firebase-auth-setup-step2-en.png"
  alt="Email/password authentication setup"
  caption="Enable email/password authentication and configure options"
/>

<DevTip>
In the initial stage, it's better to disable the "Email link" option and use standard password authentication. Users are more familiar with it.
</DevTip>

</Step>

<Step number={3} title="Add Google Sign-In (Optional)">

1. Select **"Google"** from the provider list
2. Click the **"Enable"** toggle
3. Select a **"Project support email"** (required)
4. Click **"Save"**

<Step number={4} title="Add Facebook Login (Optional)">

Allow users to sign in with Facebook accounts.

1. Select **"Facebook"** from the provider list
2. Click the **"Enable"** toggle
3. Enter **"App ID"** and **"App Secret"** (requires Facebook Developer account)
   - Create an app at Facebook Developers (https://developers.facebook.com)
   - Copy ID and Secret from app settings
4. Verify **"Authorized redirect URIs"** (auto-generated by Firebase)
5. Click **"Save"**

<Step number={5} title="User Management and Security Settings">

Complete your Authentication setup and configure additional security options.

1. Check the user list in the **"Users"** tab
2. Configure password policies in the **"Settings"** tab:
   - Set minimum password length
   - Third-party cookie policy
3. Customize **"User actions"**:
   - Email verification message
   - Password reset message
   - New user sign-up message

## Troubleshooting

<DevTip>
**"Authentication failed" error on login**

If you see this error:
1. Verify Authentication is enabled in Firebase Console
2. Check that the user account appears in the Users tab
3. Verify email and password are correct
4. Check CORS configuration
5. Review Firebase security rules (especially Firestore and Storage)
</DevTip>

<Callout type="warning">
‚ö†Ô∏è **Security Recommendations**

1. **Client-side**: Never hardcode API keys in your code
2. **Passwords**: Recommend strong passwords to users
3. **HTTPS**: Always use HTTPS
4. **Sessions**: Manage session tokens securely
5. **Logout**: Force logout after sensitive operations
</Callout>
