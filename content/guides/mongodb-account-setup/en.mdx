<Step number={3} title="Create Your First Database Cluster">

1. Click **Create** button in the dashboard.
2. **Select your plan**:
   - Choose **M0 Free** (this is the free tier)
   - No payment method required
3. **Choose cloud provider and region**:
   - AWS, Azure, or GCP

4. Click **Create Cluster** (takes 1-3 minutes).

<Step number={4} title="Create Database User and Get Connection String">

Once your cluster is created:

1. Click **Database Access** in the left sidebar.
2. Click **Add New Database User**.
3. Enter **username** (e.g., "dbuser").
4. Enter **password** (save this securely!).
5. Choose **Database User** role.
6. Click **Add User**.
7. Go to **Network Access** > **Add IP Address**.
8. Select **Allow access from anywhere** (for development).

## Frequently Asked Questions

<Callout type="info">
**Q. What's the difference between MongoDB and SQL databases?**

A. MongoDB stores flexible JSON documents (NoSQL), while SQL databases use rigid tables. MongoDB is better for flexible schemas and rapid development. SQL is better for strict data structures and complex joins.
</Callout>

<Callout type="info">
**Q. How much can I store on the free tier?**

A. 512 MB of data. This is enough for development, testing, and small projects. Once you run out, you can upgrade to a paid tier or delete old data.
</Callout>

<Callout type="info">
**Q. Can I use MongoDB for production?**

A. Yes! MongoDB Atlas is used by thousands of production apps. The free M0 tier is best for development. For production traffic, upgrade to paid tiers (M10, M20, etc.).
</Callout>

<Callout type="info">
**Q. What if I lose my password?**

A. You can reset your password in MongoDB Atlas under Database Access. Create a new user with a new password and delete the old one.
</Callout>

## Troubleshooting

### Verification email not received

- Check spam/junk folder
- Wait a few minutes for email delivery
- Click "Resend verification email" on signup page
- Try signing up again with a different email

### Cluster creation is slow

- Cluster provisioning can take 1-3 minutes
- Don't refresh the page - wait for it to complete
- If stuck over 5 minutes, try refreshing the page

### Can't connect to database

- Verify your connection string is correct
- Check that network access is allowed
- Ensure your database user exists and password is correct
- For production, configure IP whitelist instead of allowing anywhere
