<Step number={2} title="Connect GitHub Repository">

1. Select **GitHub** as your source.
2. Authorize DigitalOcean to access your GitHub account.
3. Select your **GitHub organization** or personal account.
4. Choose your **repository**.
5. Select the **branch** to deploy (usually "main" or "master").
6. Click **Next**.

  src="https://res.cloudinary.com/demo/image/upload/v1/digitalocean/app-step2-github-en.png"
  alt="GitHub repository connection"
  caption="Connect your GitHub repository to App Platform"
/>

<DevTip>
Make sure your repository is public or DigitalOcean has access via GitHub authorization. Private repositories require proper OAuth setup.
</DevTip>

</Step>

<Step number={3} title="Configure App Settings">

1. **Choose your application type**:
   - Web Service (default)
   - Static Site (HTML, CSS, JS)
   - Workers (background jobs)
2. **Set environment variables**:
   - Database URLs
   - API keys
   - Other sensitive data
3. **Configure build command** (if needed):
   - For Node.js: `npm install && npm run build`
   - For Python: `pip install -r requirements.txt`
4. **Configure run command**:
   - For Node.js: `npm start`
   - For Python: `gunicorn app:app`
5. Click **Next**.

## Frequently Asked Questions

<Callout type="info">
**Q. Does it cost money after the trial?**

A. Yes. After your $200 trial credits expire (in 60 days), App Platform charges $5-10 per month depending on your plan. You can delete your app to stop charges.
</Callout>

<Callout type="info">
**Q. How do automatic deployments work?**

A. When you push code to GitHub, a webhook notifies App Platform. App Platform automatically builds and deploys your code. Usually takes 2-5 minutes.
</Callout>

<Callout type="info">
**Q. Can I use my own domain?**

A. Yes! Add your domain in the Domains tab. You'll need to update your domain's DNS records to point to DigitalOcean.
</Callout>

<Callout type="info">
**Q. What languages does App Platform support?**

A. Node.js, Python, Go, Ruby, Java, PHP, and more. It detects your language from package.json, requirements.txt, etc.
</Callout>

## Troubleshooting

### Build failed

- Check your build command in configuration
- Verify all dependencies are in package.json or requirements.txt
- Check app logs for specific error messages
- Common issue: Missing environment variables

### App not deploying on push

- Verify GitHub authorization is still valid
- Check that you're pushing to the correct branch
- View deployment logs in App Platform
- Trigger manual redeploy from dashboard

### App deployed but showing error

- Check app logs for runtime errors
- Verify environment variables are set
- Ensure database connections are correct
- Test locally before pushing to GitHub
