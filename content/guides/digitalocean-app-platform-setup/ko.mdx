# DigitalOcean App Platform 설정 - 앱 자동 배포

DigitalOcean App Platform은 **GitHub에서 직접 애플리케이션을 배포**하고 서버를 관리하지 않도록 합니다. 코드를 push할 때마다 앱이 자동으로 배포됩니다. 30분 안에 자동 배포를 설정합니다.

## 시작하기 전에

<PrerequisiteCheck items={[
  { text: "크레딧이 있는 DigitalOcean 계정", required: true },
  { text: "저장소가 있는 GitHub 계정", required: true },
  { text: "애플리케이션 코드 (Node.js, Python 등)", required: true }
]} />

<Callout type="info">
🚀 **App Platform이란?** DigitalOcean App Platform은 Platform-as-a-Service (PaaS)입니다. 코드를 GitHub에 푸시하면 App Platform이 자동으로 빌드하고 배포합니다. 서버 관리가 필요 없습니다.
</Callout>

---

<Step number={1} title="DigitalOcean에서 새 앱 생성">

1. **DigitalOcean 제어판**에 로그인합니다.
2. 좌측 사이드바의 **앱**을 클릭합니다.
3. **앱 생성** 버튼을 클릭합니다.
4. **앱 생성 마법사**를 볼 것입니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/digitalocean/app-step1-create-app-ko.png"
  alt="DigitalOcean App Platform 생성"
  caption="DigitalOcean에서 앱 생성 시작"
  priority={true}
/>

<Callout type="tip">
💡 **팁**: App Platform은 코드가 GitHub에 있어야 합니다. 저장소가 공개이거나 GitHub 계정을 연결했는지 확인합니다.
</Callout>

</Step>

---

<Step number={2} title="GitHub 저장소 연결">

1. 소스로 **GitHub**을 선택합니다.
2. GitHub 계정에 접근하도록 DigitalOcean을 인증합니다.
3. **GitHub 조직** 또는 개인 계정을 선택합니다.
4. **저장소**를 선택합니다.
5. 배포할 **브랜치** (보통 "main" 또는 "master")를 선택합니다.
6. **다음**을 클릭합니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/digitalocean/app-step2-github-ko.png"
  alt="GitHub 저장소 연결"
  caption="GitHub 저장소를 App Platform에 연결합니다"
/>

<DevTip>
저장소가 공개이거나 DigitalOcean이 GitHub 인증을 통해 접근할 수 있는지 확인합니다. 비공개 저장소는 적절한 OAuth 설정이 필요합니다.
</DevTip>

</Step>

---

<Step number={3} title="앱 설정 구성">

1. **애플리케이션 유형** 선택:
   - 웹 서비스 (기본값)
   - 정적 사이트 (HTML, CSS, JS)
   - 워커 (백그라운드 작업)
2. **환경 변수 설정**:
   - 데이터베이스 URL
   - API 키
   - 기타 민감한 데이터
3. **빌드 명령 구성** (필요한 경우):
   - Node.js: `npm install && npm run build`
   - Python: `pip install -r requirements.txt`
4. **실행 명령 구성**:
   - Node.js: `npm start`
   - Python: `gunicorn app:app`
5. **다음**을 클릭합니다.

<Step number={4} title="앱 이름 및 요금제 설정">

1. **앱 이름**을 입력합니다 (예: "my-awesome-app").
2. **지역**을 선택합니다 (사용자와 가장 가까운 곳을 선택합니다).
3. **요금제** 선택:
   - 무료 요금제 ($200 크레딧 포함)
   - 스타터 요금제 (월 $12)
4. 구성을 검토합니다.
5. **리소스 생성**을 클릭합니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/digitalocean/app-step4-plan-ko.png"
  alt="앱 이름 및 요금제 선택"
  caption="앱 이름, 지역 및 요금제 선택"
/>

<Callout type="info">
⏳ **배포 시간**: 첫 배포는 5-10분이 걸립니다. DigitalOcean이 자동으로 앱을 빌드하고 배포합니다.
</Callout>

</Step>

---

<Step number={5} title="배포된 앱 보기 및 커스텀 도메인 설정">

1. 배포가 완료되면 앱의 **라이브 URL**을 볼 것입니다.
2. **라이브 앱** 링크를 클릭하여 배포된 애플리케이션을 봅니다.
3. (선택사항) **커스텀 도메인** 추가:
   - **도메인** 탭 클릭
   - **도메인 추가** 클릭
   - 도메인 입력 (예: myapp.com)
   - DNS 지시사항 따르기
4. 앱이 라이브이고 GitHub에 푸시할 때마다 **자동으로 재배포**됩니다!

## 다음 단계

1. **GitHub에 코드 푸시** - 자동 배포가 일어나는 것을 봅니다
2. **커스텀 도메인 추가** - 자신의 도메인 이름 사용
3. **데이터베이스 설정** - PostgreSQL 또는 MySQL 연결
4. **로그 보기** - 기본 로깅으로 문제 디버깅

## 자주 묻는 질문

<Callout type="info">
**Q. 체험 후에 비용이 드나요?**

A. 네. $200 체험 크레딧이 만료된 후 (60일 후), App Platform은 요금제에 따라 월 $5-10을 청구합니다. 앱을 삭제하여 비용 청구를 멈출 수 있습니다.
</Callout>

<Callout type="info">
**Q. 자동 배포는 어떻게 작동하나요?**

A. GitHub에 코드를 푸시하면 웹훅이 App Platform에 알립니다. App Platform이 자동으로 코드를 빌드하고 배포합니다. 보통 2-5분이 걸립니다.
</Callout>

<Callout type="info">
**Q. 자신의 도메인을 사용할 수 있나요?**

A. 네! 도메인 탭에 도메인을 추가합니다. 도메인의 DNS 레코드를 DigitalOcean으로 가리키도록 업데이트해야 합니다.
</Callout>

<Callout type="info">
**Q. App Platform은 어떤 언어를 지원하나요?**

A. Node.js, Python, Go, Ruby, Java, PHP 등입니다. package.json, requirements.txt 등에서 언어를 감지합니다.
</Callout>

## 문제 해결

### 빌드 실패

- 구성의 빌드 명령을 확인합니다
- 모든 의존성이 package.json 또는 requirements.txt에 있는지 확인합니다
- 앱 로그에서 특정 오류 메시지를 확인합니다
- 일반적인 문제: 환경 변수 누락

### 푸시에서 앱이 배포되지 않음

- GitHub 인증이 여전히 유효한지 확인합니다
- 올바른 브랜치에 푸시하고 있는지 확인합니다
- App Platform에서 배포 로그를 봅니다
- 대시보드에서 수동 재배포 트리거

### 앱이 배포되었지만 오류 표시

- 앱 로그에서 런타임 오류를 확인합니다
- 환경 변수가 설정되어 있는지 확인합니다
- 데이터베이스 연결이 올바른지 확인합니다
- GitHub에 푸시하기 전에 로컬에서 테스트하세요
