<Step number={1} title="Prepare GitHub Repository">

Prepare your GitHub repository for GitHub Actions setup.

1. Sign in to [GitHub](https://github.com)
2. Navigate to your repository
3. Verify repository structure:
   - Source code (src/, app/, etc.)
   - Package manager files (package.json, requirements.txt, etc.)
4. Prepare to create `.github/workflows` directory if needed

<Step number={2} title="Create Workflow File">

Create a YAML file that defines your GitHub Actions workflow.

1. On your repository page, click the **"Actions"** tab
2. Click the **"New workflow"** button
3. Click **"set up a workflow yourself"**
4. Set the file name to `ci.yml` or your preferred name
5. The workflow template editor opens

  src="https://res.cloudinary.com/demo/image/upload/v1/github/github-actions-workflow-setup-step2-en.png"

/>

<DevTip>
Start by modifying a basic template rather than writing from scratch to avoid syntax complexity.

</Step>

<Step number={3} title="Write Basic Workflow YAML">

Write a basic workflow for a Node.js project.

<CopyBlock
  code="name: CI Workflow

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
    - uses: actions/checkout@v3

    - name: Set up Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'

    - name: Install dependencies
      run: npm install

    - name: Run linter
      run: npm run lint

    - name: Run tests
      run: npm test

    - name: Build project
      run: npm run build"
  language="yaml"
/>

Workflow structure:
- **name**: Workflow name
- **on**: Trigger conditions (push, pull_request, etc.)
- **jobs**: List of jobs to run
- **runs-on**: Execution environment (ubuntu-latest, windows-latest, etc.)
- **steps**: Individual job steps (checkout, setup, test, etc.)

<Callout type="warning">
‚ö†Ô∏è **YAML Syntax Notes**

- **Indentation**: 2 or 4 spaces (no tabs)
- **Colon**: Space required between key and value (key: value)
- **Arrays**: Lines starting with "-" (- item)

<Step number={4} title="Commit and Run Workflow">

Save your workflow and test it.

1. In the editor, click **"Commit changes"**
2. Enter your **Commit message**:
   - Example: "Add GitHub Actions CI workflow"
3. Click **"Commit changes"** to save to repository
4. Workflow runs automatically
5. Check execution status in **"Actions"** tab:
   - Yellow dot: Running
   - Green check: Success
   - Red X: Failed

  src="https://res.cloudinary.com/demo/image/upload/v1/github/github-actions-workflow-setup-step4-en.png"

## Troubleshooting

<DevTip>
**"Workflow failed - npm ERR!"**

Check the following:
1. Verify package.json is in repository root
2. Verify required commands in package.json scripts
3. Check dependency versions are compatible
4. Explicitly set Node.js version
5. Test `npm install && npm test` locally
</DevTip>

<Callout type="warning">
‚ö†Ô∏è **Usage Time Considerations**

- **2,000 min/month limit**: Workflow fails if exceeded
- **Time calculation**: Only actual execution time counted (not waiting)
- **Private repos**: 3,000 minutes provided
- **Self-hosted runners**: No time limits
- **Prevent overuse**: Limit schedule trigger frequency
</Callout>

## Common Workflow Patterns

**Python Project:**
```yaml
- uses: actions/setup-python@v4
  with:
    python-version: '3.10'
- run: pip install -r requirements.txt
- run: python -m pytest
```

**Docker Build:**
```yaml
- uses: docker/build-push-action@v4
  with:
    context: .
    push: false
    tags: myapp:latest
```

**Environment Variables:**
```yaml
env:
  NODE_ENV: test
  API_KEY: \${{ secrets.API_KEY }}
```
