<Step number={1} title="Install with Docker (Recommended)">

Docker makes it easy to get started without dependency issues.

### Verify Docker is Installed

Open a terminal and run:

<CopyBlock language="bash" title="Check Docker version">
docker --version
</CopyBlock>

If Docker is not installed, download it from the [Docker official website](https://www.docker.com/products/docker-desktop).

### Run Elasticsearch with Docker

<CopyBlock language="bash" title="Start Elasticsearch Docker container">
docker run -d \
  --name elasticsearch \
  -e discovery.type=single-node \
  -e xpack.security.enabled=false \
  -p 9200:9200 \
  -p 9300:9300 \
  docker.elastic.co/elasticsearch/elasticsearch:8.13.0
</CopyBlock>

Command explanation:
- `-d`: Run in background
- `--name elasticsearch`: Container name
- `-e discovery.type=single-node`: Single-node mode (development)
- `-e xpack.security.enabled=false`: Disable security (development)
- `-p 9200:9200`: REST API port
- `-p 9300:9300`: Node communication port

<Step number={2} title="Manual Installation (Optional)">

You can install Elasticsearch directly without Docker.

### Download Elasticsearch

1. Visit the [Elastic Download Page](https://www.elastic.co/downloads/elasticsearch)
2. Select the latest **Elasticsearch** version
3. Download the version for your operating system (macOS, Linux, Windows)

### Install

For Linux/Mac:

<CopyBlock language="bash" title="Extract and install Elasticsearch">
cd ~/Downloads
tar -xzf elasticsearch-8.13.0-linux-x86_64.tar.gz
cd elasticsearch-8.13.0
</CopyBlock>

For Windows, download the ZIP file and extract it.

<Step number={3} title="Start the Elasticsearch Server">

### With Docker

<CopyBlock language="bash" title="Check container status">
docker ps | grep elasticsearch
</CopyBlock>

Verify the container is running. You should see `Up` status.

### With Manual Installation

<CopyBlock language="bash" title="Start Elasticsearch server">
./bin/elasticsearch
</CopyBlock>

Or run in the background:

<CopyBlock language="bash" title="Run in background">
./bin/elasticsearch -d
</CopyBlock>

When the server starts, you'll see:
```
[INFO ][elasticsearch.node] [node-name] started
```

<Step number={4} title="Verify Installation">

Test that Elasticsearch is working by accessing its API.

### Verify with curl

In your terminal:

<CopyBlock language="bash" title="Check Elasticsearch status">
curl http://localhost:9200
</CopyBlock>

If successful, you'll see a response like:

<CopyBlock language="json" title="Response example">
{
  "name" : "node-name",
  "cluster_name" : "elasticsearch",
  "cluster_uuid" : "uuid",
  "version" : {
    "number" : "8.13.0",
    "build_flavor" : "default"
  },
  "tagline" : "You Know, for Search"
}
</CopyBlock>

### Verify with Web Browser

Open your browser and visit:
```
http://localhost:9200/_cat/health
```

You'll see the cluster health status.

<Step number={5} title="Optional: Install Kibana (Monitoring Tool)">

Kibana is a visualization and management tool for Elasticsearch.

### Run Kibana with Docker

<CopyBlock language="bash" title="Start Kibana container">
docker run -d \
  --name kibana \
  -p 5601:5601 \
  -e ELASTICSEARCH_HOSTS=http://elasticsearch:9200 \
  --network bridge \
  docker.elastic.co/kibana/kibana:8.13.0
</CopyBlock>

Then open your browser:
```
http://localhost:5601
```

You'll access the Kibana dashboard.

## Troubleshooting

### Port 9200 Already in Use

Change to a different port:

<CopyBlock language="bash" title="Use different port">
docker run -d \
  --name elasticsearch \
  -e discovery.type=single-node \
  -e xpack.security.enabled=false \
  -p 9201:9200 \
  docker.elastic.co/elasticsearch/elasticsearch:8.13.0
</CopyBlock>

Then access at `http://localhost:9201`.

### Out of Memory Error

Elasticsearch allocates 2GB RAM by default. Adjust with environment variables:

<CopyBlock language="bash" title="Adjust memory allocation">
docker run -d \
  --name elasticsearch \
  -e discovery.type=single-node \
  -e xpack.security.enabled=false \
  -e "ES_JAVA_OPTS=-Xms512m -Xmx512m" \
  -p 9200:9200 \
  docker.elastic.co/elasticsearch/elasticsearch:8.13.0
</CopyBlock>

### Check Docker Logs

<CopyBlock language="bash" title="View Docker logs">
docker logs elasticsearch
</CopyBlock>

Check error messages to diagnose issues.

### Firewall Issues

Port 9200 might be blocked by firewall. For local development on localhost, this usually isn't an issue.
