# Vercel 계정 생성 및 설정 가이드

Vercel은 Next.js를 개발한 회사로, 프론트엔드 프레임워크를 위한 최고의 배포 플랫폼을 제공합니다. 이 가이드에서는 GitHub 계정으로 Vercel 계정을 생성하고, 대시보드를 둘러보며, 무료 Hobby 플랜의 혜택을 알아봅니다.

## 준비사항

시작하기 전에 다음이 필요합니다:

- GitHub 계정 (없으면 [github.com](https://github.com)에서 무료로 생성)
- 이메일 주소
- 웹 브라우저

<Callout type="info">
Vercel은 GitHub, GitLab, Bitbucket 계정으로 로그인할 수 있지만, GitHub 연동을 가장 권장합니다. Next.js와의 통합이 가장 원활합니다.
</Callout>

---

<Step number={1} title="Vercel 웹사이트 접속 및 GitHub로 회원가입">

1. 웹 브라우저에서 [vercel.com](https://vercel.com)에 접속합니다.
2. 우측 상단의 **Sign Up** 버튼을 클릭합니다.
3. 로그인 옵션 중 **Continue with GitHub**를 선택합니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/vercel/step1-signup-ko.png"
  alt="Vercel 회원가입 페이지에서 GitHub로 로그인 선택"
  caption="Continue with GitHub 버튼을 클릭하여 GitHub 계정으로 회원가입"
/>

<Callout type="tip">
GitHub 계정으로 가입하면 나중에 Git 저장소 연동이 자동으로 이루어져 배포 과정이 훨씬 간편합니다.
</Callout>

</Step>

---

<Step number={2} title="GitHub OAuth 인증 및 권한 승인">

1. GitHub 로그인 페이지가 나타나면 GitHub 사용자 이름과 비밀번호를 입력합니다.
2. **Authorize Vercel** 버튼을 클릭하여 Vercel이 GitHub 계정에 접근할 수 있도록 권한을 부여합니다.
3. 요청되는 권한을 확인합니다:
   - 저장소 읽기 권한
   - 웹훅 생성 권한 (자동 배포용)
   - 커밋 상태 업데이트 권한

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/vercel/step2-github-auth-ko.png"
  alt="GitHub OAuth 권한 승인 화면"
  caption="Vercel이 요청하는 GitHub 권한을 검토하고 승인"
/>

<Callout type="warning">
권한 승인 시 "Grant access to all repositories" 또는 "Only select repositories" 중 선택할 수 있습니다. 처음에는 특정 저장소만 선택하는 것을 권장합니다.
</Callout>

<Step number={3} title="계정 유형 선택 (개인 vs 팀)">

GitHub 인증 후, Vercel은 계정 유형을 선택하라고 요청합니다.

### 개인 계정 (Personal Account)
- 개인 프로젝트와 사이드 프로젝트에 적합
- Hobby 플랜 무료 사용
- 1명의 사용자만 접근
- 무제한 배포 가능

### 팀 계정 (Team Account)
- 협업 프로젝트에 적합
- 여러 팀원 초대 가능
- Pro 플랜부터 시작 (유료)
- 역할 기반 접근 제어

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/vercel/step3-account-type-ko.png"
  alt="Vercel 계정 유형 선택 화면"
  caption="개인 계정 또는 팀 계정 중 선택"
/>

**초보자는 개인 계정(Hobby Plan)으로 시작하는 것을 권장합니다.**

1. **Continue with Personal Account**를 선택합니다.
2. (선택사항) 계정 이름을 설정합니다. 기본값은 GitHub 사용자명입니다.

<DevTip title="팀 계정이 필요한 경우">
나중에 설정에서 언제든지 팀을 생성할 수 있습니다. 처음에는 개인 계정으로 시작하고, 협업이 필요할 때 팀으로 업그레이드하는 것이 좋습니다.
</DevTip>

</Step>

<Step number={4} title="대시보드 둘러보기 및 무료 플랜 확인">

계정 생성이 완료되면 Vercel 대시보드로 이동합니다.

### 대시보드 주요 구성요소

1. **Overview 탭**: 모든 프로젝트 목록과 최근 배포 현황
2. **New Project 버튼**: Git 저장소를 연결하여 새 프로젝트 배포
3. **Storage**: Vercel KV, Postgres, Blob 스토리지 접근
4. **Settings**: 계정 설정, 도메인, 환경 변수, 팀 관리

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/v1/vercel/step4-dashboard-ko.png"

/>

### Hobby 플랜 무료 혜택 확인

우측 상단의 계정 메뉴를 클릭하고 **Settings** > **Billing**으로 이동하면 현재 플랜을 확인할 수 있습니다.

**Hobby 플랜 무료 혜택:**
- ✅ 무제한 배포 (프로덕션 + 프리뷰)
- ✅ 월 100GB 대역폭
- ✅ 자동 HTTPS (SSL 인증서)
- ✅ 무료 `.vercel.app` 도메인
- ✅ 글로벌 CDN
- ✅ 자동 미리보기 배포 (Pull Request마다)
- ✅ Edge Functions 100GB-시간/월

## 문제 해결

### GitHub 권한 승인 실패
- GitHub에서 이미 로그인되어 있는지 확인하세요.
- 팝업 차단이 활성화되어 있다면 해제하세요.
- 시크릿 모드나 다른 브라우저로 시도해보세요.

### 이메일 인증 메일을 받지 못했어요
- 스팸 폴더를 확인하세요.
- 대시보드 우측 상단 프로필 > Settings > General에서 "Resend Verification Email"을 클릭하세요.

### 계정 유형을 잘못 선택했어요
- Settings > General에서 "Create Team" 버튼으로 언제든지 팀을 생성할 수 있습니다.
- 팀 계정은 삭제할 수 없으므로 신중하게 선택하세요.
