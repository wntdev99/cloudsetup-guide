# GCP 프로젝트 만들기 - 첫 번째 프로젝트 생성

GCP에서 모든 작업은 **프로젝트** 단위로 이루어집니다. API를 사용하거나, VM을 만들거나, 데이터베이스를 생성하려면 먼저 프로젝트가 필요합니다. 이 가이드를 따라 5분 안에 첫 번째 프로젝트를 만들어보세요!

## 프로젝트란 무엇인가요?

<Callout type="info">
📦 **프로젝트는 리소스의 컨테이너입니다**

GCP 프로젝트는 다음과 같은 역할을 합니다:
- **리소스 그룹화**: VM, 데이터베이스, 스토리지 등을 하나의 단위로 관리
- **권한 관리**: 프로젝트별로 팀원에게 접근 권한 부여
- **비용 추적**: 프로젝트별로 비용을 분리하여 확인
- **API 활성화**: 프로젝트마다 독립적으로 API를 활성화

예를 들어:
- `my-blog-project`: 블로그 웹사이트용
- `ai-experiment`: AI/ML 실험용
- `company-website`: 회사 홈페이지용
</Callout>

<Step number={1} title="프로젝트 선택 메뉴 열기">

1. [Google Cloud Console](https://console.cloud.google.com)에 로그인합니다.
2. 상단 메뉴바에서 **프로젝트 선택** 드롭다운을 클릭합니다.
   - "My First Project" 또는 "프로젝트 선택"이라고 표시됩니다.
3. 열린 팝업에서 **새 프로젝트(NEW PROJECT)** 버튼을 클릭합니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/gcp-project-create-step1-ko.png"
  alt="프로젝트 선택 드롭다운"
  caption="상단의 프로젝트 드롭다운에서 '새 프로젝트'를 선택하세요"
/>

<Callout type="tip">
💡 **팁**: 프로젝트 선택 버튼을 찾기 어렵다면, 상단 검색창에 "새 프로젝트"를 입력해보세요.
</Callout>

</Step>

---

<Step number={2} title="프로젝트 정보 입력">

### 2-1. 프로젝트 이름

1. **프로젝트 이름(Project name)** 필드에 원하는 이름을 입력합니다.
   - 예: "My First GCP Project"
   - 예: "AI Experiments"
   - 예: "Personal Blog"

<Callout type="info">
✨ **프로젝트 이름 vs 프로젝트 ID**

- **프로젝트 이름**: 사람이 읽기 쉬운 이름 (변경 가능)
- **프로젝트 ID**: GCP 전체에서 고유한 식별자 (변경 불가)

프로젝트 이름을 입력하면 프로젝트 ID가 자동으로 생성됩니다.
</Callout>

### 2-2. 프로젝트 ID (선택사항)

자동으로 생성된 프로젝트 ID를 사용하거나, **수정(EDIT)** 버튼을 클릭하여 직접 입력할 수 있습니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/gcp-project-create-step2-ko.png"
  alt="프로젝트 이름 및 ID 입력"
  caption="프로젝트 이름을 입력하면 ID가 자동 생성됩니다"
/>

<DevTip>
**프로젝트 ID 선택 팁**

- 소문자, 숫자, 하이픈(-)만 사용 가능
- 6~30자
- 전 세계적으로 고유해야 함 (이미 사용 중이면 빨간색 경고)
- 한번 생성하면 변경 불가
- 나중에 기억하기 쉬운 이름으로 설정하세요

예시:
- `my-first-gcp-2024` ✅
- `ai-experiments-john` ✅
- `MyProject123` ❌ (대문자 사용 불가)
</DevTip>

### 2-3. 조직 (선택사항)

- **조직**: 회사나 팀이 있다면 선택, 개인은 "조직 없음" 선택
- **위치**: 조직이 없으면 자동으로 "조직 없음"으로 설정됨

</Step>

---

<Step number={3} title="프로젝트 생성 및 결제 계정 연결">

1. **만들기(CREATE)** 버튼을 클릭합니다.
2. 프로젝트 생성에 몇 초 정도 소요됩니다.
3. 생성이 완료되면 자동으로 새 프로젝트로 전환됩니다.

<Screenshot
  src="https://res.cloudinary.com/demo/image/upload/gcp-project-create-step3-ko.png"
  alt="프로젝트 생성 완료"
  caption="프로젝트가 생성되고 대시보드로 이동합니다"
/>

### 결제 계정 연결 확인

프로젝트를 만든 후 결제 계정이 연결되었는지 확인하세요:

1. 왼쪽 메뉴에서 **결제(Billing)** 클릭
2. "이 프로젝트에는 결제 계정이 없습니다"라는 메시지가 보이면:
   - **결제 계정 연결** 버튼 클릭
   - 무료 체험 계정 선택
   - **계정 설정** 클릭

<Callout type="warning">
⚠️ **결제 계정이 없으면 API를 사용할 수 없습니다**

GCP의 대부분 서비스는 결제 계정이 연결되어야 사용 가능합니다. 무료 크레딧이 있더라도 결제 계정 연결은 필수입니다.
</Callout>

## 프로젝트 관리 팁

### 여러 프로젝트 만들기

목적에 따라 프로젝트를 분리하는 것이 좋습니다:

```
개인 프로젝트 구조 예시:

📦 learning-gcp (학습용)
  - Vision API 실험
  - Cloud Functions 테스트

📦 personal-blog (블로그)
  - Cloud Run 웹서버
  - Cloud Storage

📦 side-project (사이드 프로젝트)
  - Firestore DB
  - Cloud Functions
```

### 프로젝트 전환하기

여러 프로젝트 사이를 빠르게 전환하려면:
1. 상단의 프로젝트 드롭다운 클릭
2. 원하는 프로젝트 선택
3. 키보드 단축키: 검색창에서 프로젝트 이름 입력

## 자주 묻는 질문

<Callout type="info">
**Q. 프로젝트를 몇 개까지 만들 수 있나요?**

A. 기본적으로 계정당 **25개**까지 만들 수 있습니다. 더 필요하면 할당량 증가를 요청할 수 있습니다.
</Callout>

<Callout type="info">
**Q. 프로젝트 ID를 변경할 수 있나요?**

A. 아니요. 프로젝트 ID는 생성 후 **변경할 수 없습니다**. 프로젝트 이름은 언제든지 변경 가능합니다.
</Callout>

<Callout type="info">
**Q. 프로젝트를 삭제하면 어떻게 되나요?**

A. 프로젝트를 삭제하면:
- 모든 리소스가 삭제됩니다
- 30일 동안 복구 가능합니다
- 30일 후에는 완전히 삭제되며 복구 불가능합니다
- 프로젝트 ID는 영구적으로 사용 불가능해집니다
</Callout>

<Callout type="info">
**Q. 프로젝트 없이 GCP를 사용할 수 있나요?**

A. 아니요. GCP의 모든 리소스는 반드시 프로젝트 안에 있어야 합니다.
</Callout>

## 문제 해결

### "프로젝트 ID가 이미 사용 중입니다"

- 프로젝트 ID는 전 세계적으로 고유해야 합니다
- 다른 ID를 선택하거나, 숫자나 하이픈을 추가하세요
- 예: `my-project` → `my-project-2024` 또는 `my-project-john`

### 프로젝트를 만들 수 없어요

- 계정당 25개 제한을 초과했는지 확인
- 사용하지 않는 프로젝트를 삭제하세요
- 또는 할당량 증가를 요청하세요

### 결제 계정을 연결할 수 없어요

- 무료 체험 계정이 활성화되었는지 확인
- 결제 계정 권한이 있는지 확인
- 브라우저를 새로고침하고 다시 시도

## 프로젝트 명명 규칙 (Best Practices)

<Callout type="tip">
📝 **좋은 프로젝트 이름 짓기**

**이름 (사람용)**:
- 명확하고 설명적으로: "My AI Project" ✅
- 모호한 이름 피하기: "Project 1" ❌

**ID (시스템용)**:
- 환경 포함: `myapp-dev`, `myapp-prod`
- 날짜 포함: `learning-gcp-2024`
- 팀/개인 포함: `blog-john`
- 짧고 간결하게: `ai-exp` ✅ vs `artificial-intelligence-experiments-project` ❌

예시:
- `learning-gcp-2024` (학습용)
- `portfolio-prod` (포트폴리오 프로덕션)
- `blog-dev` (블로그 개발 환경)
- `ai-research-personal` (개인 AI 연구)
</Callout>
