## What is a Project?

<Callout type="info">
üì¶ **Projects are containers for resources**

A GCP project serves these purposes:
- **Resource grouping**: Manage VMs, databases, storage as one unit
- **Access control**: Grant team members permissions per project
- **Cost tracking**: View costs separately by project
- **API management**: Enable APIs independently per project

Examples:
- `my-blog-project`: For blog website
- `ai-experiment`: For AI/ML experiments
- `company-website`: For company homepage
</Callout>

### Verify Billing Account Connection

After creating your project, verify the billing account is linked:

1. Click **Billing** in the left menu
2. If you see "This project has no billing account":
   - Click **LINK A BILLING ACCOUNT**
   - Select your free trial account
   - Click **SET ACCOUNT**

<Callout type="warning">

## Frequently Asked Questions

<Callout type="info">
**Q. How many projects can I create?**

A. By default, you can create up to **25 projects** per account. You can request a quota increase if needed.
</Callout>

<Callout type="info">
**Q. Can I change the project ID?**

A. No. Project ID **cannot be changed** after creation. Project name can be changed anytime.
</Callout>

<Callout type="info">
**Q. What happens when I delete a project?**

A. When you delete a project:
- All resources are deleted
- Can be recovered for 30 days
- After 30 days, permanently deleted and cannot be recovered
- Project ID becomes permanently unusable
</Callout>

<Callout type="info">
**Q. Can I use GCP without a project?**

A. No. All GCP resources must exist within a project.
</Callout>

## Troubleshooting

### "Project ID already in use"

- Project IDs must be globally unique
- Choose a different ID or add numbers/hyphens
- Example: `my-project` ‚Üí `my-project-2024` or `my-project-john`

### Can't create project

- Check if you've exceeded the 25 project limit
- Delete unused projects
- Or request quota increase

### Can't link billing account

- Verify free trial account is active
- Check you have billing account permissions
- Refresh browser and retry

## Project Naming Best Practices

<Callout type="tip">
üìù **Choosing Good Project Names**

**Name (for humans)**:
- Clear and descriptive: "My AI Project" ‚úÖ
- Avoid vague names: "Project 1" ‚ùå

**ID (for systems)**:
- Include environment: `myapp-dev`, `myapp-prod`
- Include date: `learning-gcp-2024`

- Keep it short: `ai-exp` ‚úÖ vs `artificial-intelligence-experiments-project` ‚ùå

Examples:
- `learning-gcp-2024` (for learning)
- `portfolio-prod` (portfolio production)
- `blog-dev` (blog development)
- `ai-research-personal` (personal AI research)
</Callout>
