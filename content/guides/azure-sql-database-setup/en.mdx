# Azure SQL Database Setup - Relational Database

Azure SQL Database is a **fully managed relational database service** based on Microsoft SQL Server. It provides enterprise-grade features like automatic backups, security updates, and high availability while keeping setup and management simple. This guide shows how to create an Azure SQL Database and configure basic settings.

<Step number={1} title="Create SQL Database Server">

Create a SQL Database server in Azure Portal.

1. Sign in to [Azure Portal](https://portal.azure.com)
2. Click **"Create a resource"**
3. Search for **"SQL Database"**
4. Select **"SQL Database"**
5. Click **"Create"**

<Step number={2} title="Configure Database Basics">

Set up basic database information.

1. Select your **Subscription** (keep default)
2. Create or select **Resource Group**
3. Enter **Database name**
   - Example: "myappdb"
   - Lowercase, numbers, hyphens only
4. Create **Server**:
   - Click **"Create new"**
   - Enter **Server name** (must be globally unique)
     - Example: "mysqlserver20260208"
   - Select **Location** (recommended: East Asia)
   - Enter **Server admin login** (e.g., sqladmin)
   - Set **Password** (meeting complexity requirements)

<Step number={3} title="Configure Compute and Storage">

Set up database performance and storage.

1. Select **Compute + Storage**:
   - **Compute tier**: "Serverless" or "Provisioned"
   - For free users: "Serverless" (auto-pause, cheaper)
   - **Service tier**: Basic (B)
   - **vCores**: 2 vCores (within free tier)
   - **Storage**: 50GB (within free tier)

2. **Backup Storage**:
   - **Redundancy**: Local redundant storage (LRS)

<Step number={4} title="Configure Firewall Rules">

Set up firewall rules to manage database access permissions.

1. Go to **"Networking"** tab
2. Select **"Public endpoint"** (access from current IP)
3. **Add firewall rules**:
   - Click **"Add current client IP address"**
   - Your IP is automatically added
4. Toggle **"Allow Azure services and resources to access this server"**
   - Needed if accessing from Azure VM or App Service
5. Click **"Save"**

<Step number={5} title="Database Connection and Queries">

Connect to the created database and run basic queries.

1. Go to **SQL Database page**
2. Check **Connection strings**:
   - **ADO.NET**: For .NET applications
   - **JDBC**: For Java applications
   - **ODBC**: For Python/Node.js applications
   - **PHP**: For PHP applications

3. Use **Query Editor**:
   - Click "Query editor" in Azure Portal
   - Sign in with server admin account and password
   - Execute SQL queries

## Troubleshooting

<DevTip>
**"Database connection failed"**

Check the following:
1. Verify firewall rules allow your current IP
2. Confirm server name and database name are correct
3. Verify admin username and password are correct
4. Check connection string format is correct
5. Test in "Query editor" in Azure Portal
</DevTip>
